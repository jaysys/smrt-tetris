# 테스트계획서

## 1. 목적
본 문서는 기능을 테스트 가능한 단위로 분해하여 테스터가 독립적으로 수행할 수 있는 절차, 기대결과, 완료조건을 정의한다.

## 2. 테스트 범위
- 랜딩과 즉시 플레이
- 코어 게임 조작
- 점수/라인/게임 오버
- 모바일 세로 레이아웃과 접근성 최소 옵션
- 모드별 규칙
- 결과 화면/재도전
- 닉네임/개인 기록/랭킹
- Daily Challenge/미션/업적
- 운영 백오피스
- 분석 이벤트와 부정행위 방지

## 3. 테스트 환경
| 항목 | 기준 |
|---|---|
| 브라우저 | Chrome 최신, Safari 최신, 모바일 Chrome, 모바일 Safari |
| 디바이스 | 데스크톱 1440x900, iPhone 13급 390x844, Android 360x800 |
| 테스트 데이터 | 게스트 사용자, 운영자 계정, 기본 챌린지 데이터 |
| 네트워크 | 정상, 느림, 일시 단절 |

## 3.1 테스트 준비 절차
1. `13_테스트데이터_명세서.md` 기준으로 계정과 시드 데이터를 준비한다.
2. `14_QA실행가이드.md` 기준으로 로컬 실행 또는 테스트 환경 URL에 접속한다.
3. 날짜 전환 테스트는 가이드의 KST 기준 절차를 따른다.
4. 네트워크 장애 테스트는 브라우저 devtools 또는 프록시 도구로 재현한다.

## 4.1 정량 판정 기준
- `즉시`는 1초 이내를 의미한다.
- `빠른 재시작`은 클릭 후 2초 이내 새 게임 시작을 의미한다.
- `HUD 즉시 반영`은 이벤트 발생 후 200ms 이내 반영을 의미한다.
- 모바일 대응 기준 캡처는 `390x844`, `360x800` 두 뷰포트에서 각각 확보한다.
- 랭킹 페이지 진입 응답은 정상 네트워크에서 2초 이내 렌더를 기준으로 판정한다.
- 첫 3판 최고 기록 강조 연출은 `축하 카피`, `결과 카드 강조`, `재도전 유도` 3요소 동시 노출을 기준으로 판정한다.

## 5. 테스트 완료 기준
- Must 요구사항에 매핑된 테스트 케이스 100% 수행
- Must 요구사항 실패 0건
- Should 요구사항 실패는 우선순위와 릴리스 영향 분석 후 승인 필요
- 모바일/데스크톱 핵심 경로에서 blocker 결함 0건
- 부정행위, 기록 검증, 보상 중복 지급 관련 결함 0건

## 6. 결함 우선순위 기준
| 등급 | 설명 |
|---|---|

| Major | 모드 규칙 오류, 재도전 불가, Daily Challenge 실패, 모바일 HUD 겹침, 광고 금지 구역 침범 |
| Minor | UI 정렬, 문구, 애니메이션 이상 |
