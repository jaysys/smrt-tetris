# 개발자전달 화면설계개발 전달세트

## 1. 목적
- 본 문서는 화면설계와 시각 자산 가이드를 개발자에게 전달할 때 어떤 문서를 어떤 순서로 보내야 하는지 고정하는 전달 세트 문서다.
- 목적은 화면 구조, 캐주얼 게임 톤, 에셋 소싱 원칙이 따로 해석되지 않도록 한 번에 묶어 전달하는 것이다.

## 2. 이번 갱신의 핵심
- 화면설계 본문과 개발자 전달 요약 문서에 `OpenGameArt 1순위`, `CC0 우선`, `Kenney 2순위`, `유틸 아이콘은 Heroicons/Material Symbols 우선` 원칙을 일관되게 반영했다.
- 따라서 이제 화면 구현은 `구조`, `감정 톤`, `시각 자산 소싱`을 분리하지 말고 하나의 개발 가이드 세트로 받아야 한다.

## 3. 개발자에게 전달할 필수 문서
1. `23_UI화면설계서.md`
   - 화면별 구조, CTA 위계, 모바일 세로 배치의 본문 기준
2. `24_개발자전달_UI화면설계_요약.md`
   - 개발 착수용 요약 인계 문서
3. `26_개발자전달_캐주얼게임_UI강화요청서.md`
   - 캐주얼 게임 서비스 톤 강화 요구 문서
4. `27_개발자전달_시각에셋_아이콘_배경_컬러가이드.md`
   - 아이콘, 배경, 버튼, 색상, 에셋 소싱 원칙 문서
5. `29_개발자전달_게임화면_조작단순화_가이드.md`
   - 좌우·하단 버튼 과다 문제를 줄이고 게임패드형 조작으로 재구성하는 문서
6. `30_개발자전달_게임폰트_UI스킨_적용지시서.md`
   - 게임형 폰트와 Kenney 프레임 적용을 고정하는 문서

## 4. 필요시 함께 전달할 보조 문서
1. `25_기획자_화면설계개선의견_설계자회신.md`
   - 왜 화면 톤을 상향했는지 배경 설명이 필요할 때 전달
2. `05_화면흐름도_및_사용자시나리오.md`
   - 화면 전환 흐름을 함께 맞춰야 할 때 전달
3. `01_기능설계서.md`
   - UI와 기능 규칙을 함께 확인해야 할 때 전달

## 5. 전달 순서
1. `24_개발자전달_UI화면설계_요약.md`
2. `26_개발자전달_캐주얼게임_UI강화요청서.md`

6. `23_UI화면설계서.md`

## 6. 개발자에게 같이 전달할 핵심 문장
- 이번 갱신부터 화면 구현 기준은 `23`, `24`, `26`, `27`, `29`, `30` 문서를 하나의 세트로 본다. 랜딩, 플레이, 결과, 랭킹은 단순 정보 배치가 아니라 캐주얼 게임 서비스 톤으로 재설계해야 하며, 플레이 화면은 하단의 이동, 드롭, 액션 온스크린 버튼을 모두 제거하고 `옆면 키보드 안내 + 상단 간단 HUD + Next 1 미니 프리뷰` 구조로 바꿔야 한다. 전역 폰트는 `Black Han Sans`, `Orbitron`, `Signika` 체계로 교체하고, 남아 있는 박스와 버튼은 `Kenney UI Pack` 계열 프레임으로 바꾼다. 실제 게임형 오브젝트와 배경 타일은 `OpenGameArt`를 1순위로 검토하고 `CC0` 자산을 우선 사용한다. 설정/공유/닫기 같은 유틸 아이콘은 `Heroicons` 또는 `Material Symbols`를 우선 적용한다.

## 7. 최종 정리
- 개발자에게 바로 전달할 기본 세트는 `23`, `24`, `26`, `27`, `29`, `30`이다.
- 설명 배경까지 같이 전달하려면 `25`, `05`, `01`을 추가한다.
