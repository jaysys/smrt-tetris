# 개발자전달 게임화면 조작단순화 가이드

## 1. 목적
- 본 문서는 현재 게임 화면이 `업무용 페이지`처럼 보이는 문제를 해결하기 위해, 플레이 화면의 버튼 구조를 게임형으로 단순화하는 구현 기준을 개발자에게 전달하는 문서다.
- 목적은 `보드 주인공`, `하단 온스크린 버튼 제거`, `옆면 키보드 안내`, `비핵심 기능 숨김` 원칙을 고정하는 것이다.

## 2. 현재 문제 정의
- 좌우와 하단에 작은 버튼이 많아 시선이 분산된다.
- 보드 밖에 기능 버튼이 흩어져 있어 플레이보다 제어판처럼 보인다.
- 설정, 안내, 상태, 실험용 기능이 같은 강도로 노출되면 게임 화면이 아니라 업무용 대시보드처럼 느껴진다.

## 3. 외부 레퍼런스 해석
- 공식 Tetris 모바일은 `quick round`, `single player`, `intuitive touch controls`와 `on-screen controls`를 전면에 둔다.
  - 출처: https://tetris.com/products/video-game/tetris
- Royal Match는 `smooth, curved forms`, `power-ups`, `boosters`, `fun, quirky`한 오브젝트를 통해 화면을 제어판이 아니라 월드 일부처럼 보이게 만든다.
  - 출처: https://www.dreamgames.com/games/royal-match
- Subway Surfers는 `best-in-class moment-to-moment gameplay`와 강한 시각 집중을 강조한다.
  - 출처: https://subway-surfers.com/

해석:
- 플레이 순간에는 `지금 필요한 조작`만 보여야 한다.
- 보드 주변은 버튼 나열보다 `상태`, `월드`, `반응`이 읽히는 구조가 적합하다.
- 버튼은 많아 보이지 않아야 하고, 눌렀을 때는 크고 명확하게 반응해야 한다.

## 4. 설계 결론
- 게임 화면 좌우에는 정보 패널만 둔다.
- 모든 상시 조작 버튼은 제거한다.
- 플레이 화면에는 어떤 키를 쓰는지만 옆면에 아주 간단히 안내한다.
- 플레이와 직접 무관한 기능은 `일시정지` 안으로 숨긴다.

## 5. 상시 노출 허용 요소
### 5.1 좌측
- 짧은 키보드 안내 1개
- 짧은 상태 피드백 1개 이하

### 5.2 우측
- Next 1 미니 프리뷰
- 점수 상승 피드백 또는 콤보 피드백

## 6. 상시 노출 금지 요소
- 설정
- 접근성 상세 옵션
- 도움말
- 랭킹 이동
- 공유
- 재시작
- 나가기
- 개발용 상태 토글
- 디버그 텍스트
- 실험 배너

위 항목은 플레이 중 상시 노출하지 않고 `일시정지 오버레이` 안으로 이동한다.

## 7. 권장 화면 구조
1. 상단 얇은 상태 바
   - 모드명
   - 작은 HUD 박스: 점수 / 라인 / 레벨 / 플레이 시간
   - 일시정지
2. 보드 중심 본체
   - 좌측 또는 우측: 키보드 안내 미니 박스
   - 반대쪽 상단: Next 1 미니 프리뷰
3. 보드 위 단기 피드백
   - Combo
   - Back-to-Back
   - Perfect Clear
4. 하단
   - 온스크린 조작 버튼 없음

## 8. 키보드 안내 규칙
- 키 안내는 1줄 또는 2줄 이내로만 보여준다.
- 예시
  - `←→ 이동 / ↑ 회전 / ↓ 내림`
  - `Space 즉시낙하 / C 보관 / Esc 일시정지`
- 설명문, 긴 도움말, 사용 가능 여부 문구는 넣지 않는다.

## 9. 시각 언어 규칙
- 좌우 정보 카드도 업무용 위젯처럼 보이지 않게, 둥근 패널과 게임형 프레임을 사용한다.
- 동일 톤 사각 박스를 반복해 `설정 페이지`처럼 보이게 만들지 않는다.
- 키보드 안내는 작고 명료해야 하며, 보드보다 더 눈에 띄면 안 된다.
- Next 프리뷰는 `딱 1개 블록 모양`만 아주 작게 보인다.
- `보관 EMPTY 사용 가능` 같은 설명형 상태 박스는 없앤다.

## 10. 구현 금지사항
- 보드 좌우에 비핵심 기능 버튼 상시 배치
- 하단 온스크린 조작 버튼 배치
- 상단, 좌측, 우측, 하단에 같은 기능을 중복 노출
- 작은 텍스트 버튼과 아이콘 버튼을 혼합해 대시보드처럼 보이게 만드는 방식
- 플레이 도중 랭킹, 공유, 설정을 메인 조작과 같은 크기로 노출하는 방식
- `보관 EMPTY 사용 가능` 같은 설명 텍스트를 상단 HUD에 넣는 방식
- Next를 여러 개 쌓아 보드보다 더 눈에 띄게 만드는 방식

## 11. 개발자 액션 요청
1. 보드 좌우의 상시 클릭형 버튼을 제거한다.
2. 비핵심 기능은 일시정지 오버레이로 이동한다.
3. 하단 이동, 드롭, 액션 온스크린 버튼을 모두 제거한다.
4. 보드 옆에는 어떤 키를 쓰는지만 아주 간단히 안내한다.
5. 상단 HUD는 `점수`, `라인`, `레벨`, `플레이 시간`만 남긴다.
6. Next는 1개만 아주 작게 보여준다.
7. 최종 화면이 `키보드 안내가 붙은 게임 화면`처럼 보여야 하며 `업무용 제어판`처럼 보이면 반려한다.

## 12. 최종 전달 문안
- 현재 게임 화면은 좌우와 하단에 버튼이 과다하게 분산되어 업무용 페이지처럼 보인다. 앞으로 플레이 화면은 보드 중심 구조를 유지하되, 하단의 이동, 드롭, 액션 온스크린 버튼을 모두 제거하고 보드 옆에 어떤 키를 쓰는지만 아주 간단히 안내해 달라. 상단에는 `점수`, `라인`, `레벨`, `플레이 시간`만 담는 작은 HUD 박스를 두고, `보관 EMPTY 사용 가능` 같은 설명형 텍스트는 없앤다. 다음 블록은 딱 1개만 아주 작게 보여준다. 최종 결과는 `버튼이 많은 화면`이 아니라 `키보드 안내가 붙은 게임 화면`처럼 보여야 한다.
