# 캐주얼게임 UI강화 검토의견서

## 1. 목적
- 본 문서는 `2.설계문서/26_개발자전달_캐주얼게임_UI강화요청서.md`를 검토한 뒤, 현재 구현 상태와의 차이, 수용 여부, 개발 영향 범위를 정리하기 위한 의견서다.

## 2. 검토 결론
- 요청 방향은 타당하다.
- 구현 가능하다.
- 다만 이번 요청은 단순 스타일 보정이 아니라 D2 UI 범위의 `전면 재디자인 요구`로 봐야 한다.

내 의견은 다음과 같다.
- 현재 구현된 `FE-008 ~ FE-011`은 구조 검증용 1차 화면으로는 유효하다.
- 하지만 `캐주얼 게임 서비스 톤` 관점에서는 아직 미완성이다.
- 따라서 현재 UI 1차 제3자 시험은 `구조/흐름 검증`으로 유지하고, 별도 `캐주얼 UI 강화` 작업을 추가 등록하는 것이 맞다.

## 3. 항목별 의견
### 3.1 랜딩 Hero 강화
- 의견: 즉시 반영 필요
- 현재 상태: 부분 충족
- 판단:
  - `바로 시작` 1등 CTA 구조는 반영됐다.
  - 그러나 Hero의 감정 밀도, 보상 언어, 시각 오브젝트는 아직 약하다.
  - 랜딩은 카드형 정보 화면에서 `플레이 충동 유도 Hero`로 더 밀어야 한다.

### 3.2 모드 선택 포스터화
- 의견: 즉시 반영 필요
- 현재 상태: 부분 충족
- 판단:
  - 현재는 카드형 설명 구조는 갖췄다.
  - 하지만 요청서가 요구한 `포스터`, `감정 톤 분리`, `대표 비주얼`, `경쟁/보상 포인트`까지는 아니다.

### 3.3 플레이 화면 게임패드형 감각
- 의견: 후속 우선 반영 필요
- 현재 상태: 부분 충족
- 판단:
  - 보드 중심 구조는 맞다.
  - 하지만 HUD와 조작부가 아직 게임 전용 인터페이스보다 일반 웹 버튼/카드에 가깝다.
  - 보드 가독성을 해치지 않는 범위에서 광원, 입자, 조작감 시각 요소를 보강해야 한다.

### 3.4 결과 Hero와 보상 언어
- 의견: 즉시 반영 필요
- 현재 상태: 부분 충족
- 판단:
  - `재도전` 1등 CTA와 Hero 대표 지표는 반영됐다.
  - 하지만 축하 상태, 배지, 자랑하고 싶은 공유 CTA, 첫 3판 PB 연출은 아직 부족하다.

### 3.5 랭킹 경쟁 무드 강화
- 의견: 즉시 반영 필요
- 현재 상태: 부분 충족
- 판단:
  - 현재는 기본 리스트 구조다.
  - 요청서 기준 `Top 3 시상대`, `메달/트로피`, `내 기록 강조 카드`, `비교 언어`가 필요하다.

### 3.6 Daily/닉네임 보상 오브젝트화
- 의견: 후속 우선 반영 필요
- 현재 상태: 부분 충족
- 판단:
  - 바텀시트 흐름 자체는 맞다.
  - 하지만 감정 카피, 보상 오브젝트, 수집 욕구 유도는 아직 부족하다.

## 5. 개발 영향
- 현재 D2 UI 1차 구현을 폐기할 필요는 없다.
- 대신 아래처럼 분리 관리해야 한다.
  - `현재 QA-004`: 구조/흐름 검증
  - `후속 UI 강화 작업`: 캐주얼 게임 서비스 톤 반영

이유:
- 현재 구현은 화면 흐름, 반응형, CTA 계층, 바텀시트 구조를 이미 갖췄다.
- 이를 버리고 처음부터 다시 만드는 것보다, 현재 구조 위에 감정/보상/경쟁 시각 계층을 덧입히는 편이 빠르고 안전하다.

- 현재 구현은 `제품 구조 1차`, 새 요청은 `캐주얼 게임 감정/보상/경쟁 강화 2차`로 구분하는 것이 가장 현실적이다.
- 따라서 지금 상태를 실패로 뒤집지 말고, 후속 UI 재설계 작업을 별도 등록해 진행하는 것이 타당하다.
