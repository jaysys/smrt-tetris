# 기능별 테스트수행 MVP

## 1. 목적
개발 단계별로 어떤 기능이 구현되면 어떤 테스트를 바로 시작할 수 있는지 연결하여, 개발팀과 테스트팀의 병행 작업 기준을 정의한다.

## 2. 단계별 개발-테스트 연결
| 단계 | 개발 완료 항목 | 즉시 수행 테스트 | 테스트 완료 기준 |
|---|---|---|---|
| MVP-T0 | 프로젝트 부트스트랩, 랜딩 기본 화면 | TC-001 | 랜딩 기본 CTA 3종 노출 |
| MVP-T1 | 게임 엔진 기본 이동/회전/드롭, Ghost Piece, 모바일 기본 레이아웃 | TC-004, TC-005, TC-006, TC-007, TC-009, TC-028, TC-028A | 핵심 조작, Ghost Piece, 모바일 기본 레이아웃 정상 |
| MVP-T2 | 라인 제거, 점수 계산, 게임 오버 | TC-010, TC-011, TC-012 | 코어 루프 완료 |
| MVP-T3 | 결과 화면, 재도전, 초기 성취감 연출 | TC-016, TC-017, TC-017A | 한 판 종료 후 재도전과 초기 성취감 연출 확인 |
| MVP-T4 | Marathon 레벨업, Sprint 완료 규칙 | TC-013, TC-014 | 모드 분기 정상 |

| MVP-T6 | 게스트 토큰, 세션 생성/종료 API, 핵심 분석 이벤트 | TC-002, TC-029, TC-034, TC-035 | 세션 생성/종료와 핵심 이벤트 적재 안정화 |
| MVP-T7 | 닉네임 등록, 개인 최고 기록 | TC-018, TC-019, TC-033 | 기록 식별 가능 |

| MVP-T9 | 공유 링크 | TC-022 | 공유 URL 접근 가능 |
| MVP-T10 | 미션/업적 | TC-024, TC-025 | 리텐션 루프 정상 |
| MVP-T11 | 설정/접근성 | TC-026, TC-027 | 접근성 최소 옵션 완료 |
| MVP-T12 | 운영 백오피스, 부정행위 검출, 광고 보호 규칙 | TC-030, TC-031, TC-032, TC-036 | 운영 가능 상태와 코어 루프 보호 확인 |

## 3. 개발자 구현 권장 순서
1. 랜딩, 게임 엔진, 결과 화면을 먼저 구현한다.
2. Marathon 단일 모드와 모바일 세로 기본 레이아웃으로 코어 루프를 완성한다.
3. 핵심 분석 이벤트를 코어 루프와 함께 붙인다.
4. Sprint와 Daily Challenge를 추가해 모드 분기를 완성한다.
5. 세션 API, 결과 제출, 닉네임, 개인 기록을 붙인다.
6. 랭킹과 공유를 붙여 경쟁 루프를 완성한다.
7. 미션, 업적, 운영 도구를 붙여 재방문 구조를 완성하고 반응형/접근성을 최종 고도화한다.

## 4. 테스터 착수 타이밍
- 게임 엔진 QA는 API 완료를 기다리지 않는다.
- 랭킹 QA는 결과 검증 API가 연결된 이후 시작한다.
- 운영 QA는 챌린지 생성, 공지, 제재 화면이 붙은 이후 시작한다.

## 5. 단계 종료 조건
| 단계 | 종료 조건 |
|---|---|
| 코어 플레이 단계 | 플레이, 점수, 게임 오버, 재도전이 수동 개입 없이 반복 가능 |
| 경쟁 루프 단계 | 공식 기록 제출과 랭킹 반영 확인 가능 |
| 리텐션 단계 | Daily, 미션, 업적이 중복 보상 없이 작동 |
| 운영 단계 | 운영자가 코드 배포 없이 챌린지와 공지를 조정 가능 |
