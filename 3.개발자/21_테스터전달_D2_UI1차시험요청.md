# D2 UI 1차 화면 제3자 시험 요청

## 1. 목적
- 본 문서는 D2 본 기능 중 UI 1차 화면 범위 `FE-008 ~ FE-011`에 대한 제3자 시험을 요청하기 위한 문서다.
- 이번 시험 범위는 `SC-02 모드 선택`, 랜딩 CTA 위계, `SC-05 결과 화면 Hero/CTA`, `SC-06 랭킹`, `SC-07/08/09` 바텀시트다.

## 2. 개발 측 반영 범위
- 랜딩
  - `바로 시작`, `모드 선택`, `랭킹` CTA 3종 1화면 노출
  - Daily 프리뷰와 개인 성과 카드를 보조 위계로 배치
- 모드 선택
  - Marathon, Sprint, Daily Challenge 카드
  - 모드별 설명, 종료 조건, 랭킹 기준, `이 모드 시작`
- 랭킹
  - 모드 탭
  - `일간`, `주간`, `전체` 세그먼트
  - 내 기록 카드와 기본 랭킹 리스트
- 결과 화면
  - Hero 대표 지표
  - 상태 배지 줄
  - `재도전`, `랭킹`, `공유`, `기록 저장` 2x2 CTA
- 바텀시트
  - 설정
  - 닉네임 등록
  - Daily 상세

## 3. 개발 측 사전 검증 결과
- 워크스페이스 루트 기준 아래 명령 Pass
  - `pnpm test`
  - `pnpm build`
  - `pnpm test:e2e`

## 4. 시험 요청 범위
- 랜딩
  - 첫 화면에서 `바로 시작`, `모드 선택`, `랭킹`이 모두 보이는지 확인
  - `바로 시작`이 가장 강한 CTA인지 확인
  - 개발 검증 문구, 내부 상태 설명이 사용자용 랜딩 첫 화면에 남지 않았는지 확인
- 모드 선택
  - `mode-button`으로 별도 화면 진입 가능 여부
  - 3개 모드 카드의 정보 구조 확인
  - 모드 선택 후 해당 모드 시작 가능 여부
- 랭킹
  - `ranking-button`으로 랭킹 화면 진입 가능 여부
  - 모드 탭, 기간 세그먼트, 내 기록 카드, 리스트 노출 확인
- 결과 화면
  - `재도전`이 가장 강한 CTA인지 확인
  - CTA 4개가 첫 화면에서 모두 보이는지 확인
  - 결과 Hero와 상태 배지가 가려지지 않는지 확인
- 바텀시트
  - 설정, 닉네임 등록, Daily 상세가 페이지 전환이 아니라 바텀시트로 열리는지 확인
  - 닫기 후 기존 화면 흐름이 유지되는지 확인

## 5. 권장 수행 절차
1. 워크스페이스 루트에서 `pnpm test` 실행
2. 1번 성공 후 `pnpm build` 실행
3. 2번 성공 후 `pnpm test:e2e` 실행
4. 브라우저에서 공개 웹을 열고 모바일 세로 `390x844`, `360x800` 기준으로 수동 확인 수행
5. 결과를 신규 테스트결과서에 기록

## 7. 판정 기준
- `pnpm test`, `pnpm build`, `pnpm test:e2e`가 모두 종료 코드 `0`이면 Pass
- 랜딩, 모드 선택, 랭킹, 결과 화면의 핵심 UI가 설계 위계대로 보이면 Pass
- 설정/닉네임/Daily 상세가 바텀시트로 열리고 기존 흐름을 끊지 않으면 Pass
- 위 항목이 모두 Pass면 `FE-008 ~ FE-011` 범위는 제3자 시험 Pass로 판정 가능

## 8. 참고 사항
- `SC-04 일시정지 오버레이`, 첫 3판 PB 강조 상태, 실제 기록 저장 연동은 이번 시험 범위에 포함하지 않는다.
