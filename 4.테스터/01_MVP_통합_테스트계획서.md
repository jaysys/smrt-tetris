## 2. 최신 설계 변경 반영 요약
- 모바일 세로 기본 레이아웃은 후순위 보강 항목이 아니라 초기 완료 조건이다.
- 접근성 최소 옵션과 핵심 분석 이벤트 5종은 D3 이후 보강이 아니라 D1~D3 품질 게이트다.
- Marathon 랭킹은 `점수 내림차순 -> 총 생존 시간 오름차순 -> 총 입력 수 오름차순 -> 먼저 달성한 기록 우선` 규칙으로 고정한다.
- 결과 화면은 첫 3판 이내 개인 최고 기록 갱신 시 `축하 카피 + 결과 카드 강조 + 재도전 유도` 3요소를 함께 검증한다.
- 광고/수익화는 `플레이 중`, `첫 결과 화면의 첫 CTA 영역`, `재도전 직전`에서 코어 루프를 침범하면 안 된다.
- 엔진 규칙은 내부 추정이 아니라 `7-bag`, `SRS`, `Hold 재정렬`, `Ghost 착지 예측`을 외부 참조 기준까지 포함해 고정 검증한다.
- 랜딩 첫 화면은 `바로 시작`, `모드 선택`, `랭킹` 3개 CTA가 모두 보이되 `바로 시작`이 가장 강한 위계를 가져야 한다.
- 게임 화면은 모바일 세로 `상단 상태 바 -> HUD 리본 -> 보드 중심 -> 상태 피드백 -> 하단 고정 조작 도크` 구조와 safe area 대응을 완료 조건으로 본다.
- 결과 화면은 Hero 결과 카드, PB/성과 배지, 2x2 CTA 그리드가 첫 화면에 함께 보이고 `재도전`이 가장 강한 CTA여야 한다.
- 닉네임 등록과 설정은 모바일에서 전체 페이지 전환보다 모달 또는 바텀시트 우선 구조를 충족해야 한다.
- 사용자 화면에는 기술 검증 카드, 내부 상태 설명, 임시 영문 문구, 개발 단계 문구가 남아 있으면 안 된다.

## 4. 테스트 목표
- 로그인 없이 30초 이내 첫 플레이가 가능한지 검증한다.
- 엔진 규칙이 `7-bag`, `SRS`, `Hold 1턴 1회`, `Hold 재호출 시 스폰 방향 재정렬`, `Ghost 착지 예측` 기준을 만족하는지 검증한다.
- 랜딩 첫 화면에서 `바로 시작`, `모드 선택`, `랭킹`이 모두 보이고, 기술 검증용 UI가 제거되었는지 검증한다.
- 모바일 세로 `390x844`, `360x800`에서 safe area를 포함해 HUD, 보드, 조작부, 결과 CTA가 가려지지 않는지 검증한다.
- 접근성 최소 옵션과 핵심 분석 이벤트 5종이 설계 명세대로 동작하는지 검증한다.
- 조작 버튼 최소 크기, 색상 외 상태 인지, 모달/바텀시트 우선 구조가 설계 명세와 일치하는지 검증한다.
- 결과 루프, 개인 최고 기록, 랭킹, 공유, Daily 보상, 운영 기능이 데이터 정합성을 유지하는지 검증한다.
- 광고 보호 규칙과 부정행위 차단 규칙이 코어 플레이 및 결과 흐름을 침범하지 않는지 검증한다.

## 5. 테스트 원칙
- 모든 판정은 설계 문서의 수치, API 계약, 외부 참조 기준을 우선 적용한다.
- 화면 판정과 API/데이터 판정을 분리해서 기록한다.
- 시간 경계 기능은 `TEST_CLOCK_ISO_UTC` 기준으로 검증한다.
- 이벤트 미충족 여부는 단일 파일이 아니라 앱 코드 및 수집 결과 전체를 기준으로 판단한다.
- 엔진 점수 검증은 일부 점수 항목이 아니라 `착지 -> 고정 -> 라인 수 계산 -> Perfect Clear -> Back-to-Back -> Combo -> 점수 계산 -> HUD 갱신 -> 다음 블록 스폰 -> 스폰 실패 시 게임 오버` 순서 전체 정합성을 본다.
- 랭킹 규칙은 `BE/DB/QA 기대결과`를 함께 잠근다.
- `DONE` 판정은 테스트 통과와 문서 현행화까지 끝난 뒤에만 가능하다.

## 10. 진입 조건과 종료 조건

- `pnpm install`, `pnpm db:migrate`, `pnpm db:seed`, `pnpm dev` 또는 동등 실행 절차가 완료되어야 한다.
- `13_테스트데이터_명세서.md` 기준 데이터가 준비되어야 한다.
- 모바일 `390x844`, `360x800` 뷰포트 검증 환경이 준비되어야 한다.
- 분석 이벤트 적재 저장소 또는 관측 경로가 준비되어야 한다.

### 10.2 테스트 종료 조건
- Must 요구사항 매핑 테스트 100% 수행 완료
- Critical 0건, Major 0건
- 코어 엔진 규칙, 랭킹 오염, 보상 중복, 광고 금지 구역 침범 결함 0건
- 모바일 `390x844`, `360x800` 핵심 경로 blocker 0건
- 이벤트 5종 적재 확인 완료
- P0 자동화 세트 통과

## 17. 최종 판정 기준
- 랜딩, 플레이, 결과, 재도전, 랭킹, Daily, 운영 핵심 경로가 모두 Pass다.
- 엔진 규칙 `7-bag`, `SRS`, `Hold`, `Ghost`, 점수 순서가 설계 기준과 일치한다.
- 이벤트 5종이 저장되고 모바일 `390x844`, `360x800`에서 blocker가 없다.
- 첫 3판 PB 강조 연출과 광고 금지 구역 보호 규칙이 모두 충족된다.
- 미해결 결함은 Minor만 존재하며 릴리스 영향 분석이 승인되었다.
