# 운영배포체크리스트

## 1. 배포 전 확인
- 브랜드/IP 정책 확정 여부 확인
- 서비스명, 로고, 주요 UI 문구 법무 검토 여부 확인
- 개인정보 처리방침, 이용약관, 쿠키 정책 배포 여부 확인
- Daily Challenge와 공지 기본 데이터 등록 여부 확인
- 운영자 계정 생성 및 권한 검증 완료 여부 확인

## 2. 인프라 체크
- 웹 앱 배포 환경 변수 적용 완료
- API 서버 DB 연결 정상
- SQLite3 DB 파일 경로 및 쓰기 권한 정상
- SQLite3 WAL 모드 적용 여부 확인
- 로그 수집 경로 정상
- HTTPS 인증서 정상
- 레이트 리밋 설정 적용

## 2.1 표준 환경변수 목록
| 변수명 | 설명 | 예시 |
|---|---|---|
| `NODE_ENV` | 실행 환경 | `production` |
| `PORT` | API 포트 | `3001` |
| `WEB_BASE_URL` | 웹 서비스 주소 | `https://game.example.com` |
| `ADMIN_BASE_URL` | 운영 페이지 주소 | `https://game.example.com/admin` |
| `SQLITE_DB_PATH` | SQLite3 파일 경로 | `./data/app.db` |
| `JWT_SECRET` | JWT 서명 키 | `change-me` |
| `GUEST_TOKEN_SECRET` | 게스트 토큰 서명 키 | `change-me-too` |
| `LOG_LEVEL` | 로그 레벨 | `info` |
| `TZ` | 프로세스 시간대 | `UTC` |

## 3. DB 체크
- 마이그레이션 최신 상태 적용
- `game_modes` 기본 데이터 존재
- 기본 미션, 업적, 금칙어 사전 적재 완료
- 관리자 초기 계정과 권한 데이터 준비

## 3.1 표준 실행 명령 규약
아래 명령은 `예정 명령 규약`이다.
현재 프로젝트 폴더에는 실제 코드 저장소와 `package.json`이 확인되지 않았으므로, 구현 저장소 생성 후 동일 이름으로 스크립트를 맞추거나 문서를 갱신해야 한다.

```bash
pnpm install
pnpm db:migrate
pnpm db:seed
pnpm dev
pnpm test
pnpm test:e2e
```

배포 시 예정 명령

```bash
pnpm build
pnpm db:migrate
pnpm db:seed
pnpm start
```

## 4. 기능 체크
- 게스트 즉시 플레이 가능
- Marathon, Sprint, Daily Challenge 시작 가능
- 결과 제출과 랭킹 반영 가능

## 6. 배포 절차
1. DB 백업 생성
2. 마이그레이션 실행
3. API 서버 배포
4. 웹 앱 배포
5. 운영 백오피스 배포
6. 선택 구성 사용 시 캐시 워밍
7. 스모크 테스트 수행

## 6.1 스모크 테스트 기준 URL
- 사용자 메인: `/`
- 모드 선택: `/modes`
- 랭킹: `/leaderboards`
- 공유 결과: `/share/{shareId}`
- 운영 로그인: `/admin/login`
- 운영 챌린지: `/admin/daily-challenges`

## 6.2 로그 확인 위치
- 애플리케이션 로그: 표준 출력 또는 `/var/log/app`
- 에러 로그: 에러 집계 도구 또는 `error` 레벨 스트림
- 부정행위 검증 로그: `fraud-validation` 태그 필터
- 배치/스케줄러 로그: `scheduler` 태그 필터

## 6.3 장애 시 우선 확인 API
- `GET /v1/bootstrap`
- `POST /v1/game-sessions`
- `POST /v1/game-sessions/{id}/finish`
- `GET /v1/leaderboards`
- `GET /v1/daily-challenge`

5. 원인 분석 후 재배포 일정 확정

## 9.1 실제 실행 순서 예시
```bash
pnpm stop
git checkout <previous-release-tag>
pnpm install --frozen-lockfile
pnpm db:migrate:rollback
pnpm start
```

주의
- 위 명령은 `예시`다.
- 실제 롤백 명령은 운영 저장소 스크립트에 맞춰 최종 확정한다.
- SQLite3 파일 백업본은 배포 직전 스냅샷으로 보관한다.
