- 문서 버전: v1.0
- 작성일: 2026-03-11
- 작성 목적: `3.개발자/screenshots` 기준 현재 화면이 기획 의도와 흥행형 캐주얼 게임 서비스 톤에 맞는지 검토하고, 설계자에게 전달할 강화 요구사항을 정리한다.
- 검토 범위: `01-landing.png`, `02-settings-sheet.png`, `03-daily-sheet.png`, `04-mode-select.png`, `05-ranking.png`, `06-playing.png`, `07-result.png`, `08-nickname-sheet.png`

## 1. 총평

현재 화면은 `구조 확인용 프로토타입`으로는 의미가 있다. 그러나 `캐주얼 게임 서비스`가 아니라 `정적인 정보 카드형 모바일 웹`처럼 보인다. 기획 의도였던 `즉시 플레이`, `짧은 재도전`, `기록 경쟁`, `데일리 재방문`, `성취감 강화`는 기능 텍스트로는 존재하지만, 시각 경험으로는 거의 전달되지 않는다.

기획자 판단은 명확하다. 현재 화면은 `서비스가 흥행하려는 게임 화면 톤`에 미달한다. 특히 아래 문제가 크다.

1. 브랜드 감정이 없다.
2. 게임다운 긴장감과 보상감이 없다.
3. 캐주얼 게임 특유의 세계관, 캐릭터, 장난기, 수집 욕구가 없다.
4. 주요 CTA가 강하지 않고, 모든 카드가 비슷하게 보여 정보 위계가 약하다.
5. 플레이 화면조차 `게임 장면`보다 `관리 패널`에 가깝다.

따라서 화면설계는 `정보 정리형 UI`가 아니라 `플레이 충동을 만들고, 다시 누르게 만들고, 보상을 기대하게 만드는 게임 서비스 UI`로 재설계되어야 한다.

## 2. 스크린샷 기준 핵심 문제

### 2.1 전반 문제

- 전체 색감이 지나치게 단조롭고, 화면 대부분이 비슷한 톤의 카드 박스로 구성되어 있다.
- 제목 타이포는 강하지만, 주변 요소가 너무 약해 화면 에너지가 죽어 있다.
- 캐릭터, 마스코트, 테마 오브젝트, 스티커형 보상 장치가 없어 감정 밀도가 낮다.
- 데일리, 랭킹, 기록, 업적처럼 사용자를 다시 오게 만드는 장치가 시각적으로 거의 보상처럼 보이지 않는다.
- `게임 서비스`가 아니라 `모바일 문서 서비스`처럼 보이는 레이아웃이다.

### 2.2 랜딩 화면 문제

- 첫 화면이 너무 조용하다. `바로 시작`을 누르고 싶게 만드는 폭발력이 부족하다.
- Hero 영역에 게임 플레이의 쾌감이나 독자 IP의 개성이 보이지 않는다.
- 데일리 챌린지, 최근 성과, 목표 등이 모두 비슷한 카드 강도로 눕혀 있어 무엇이 중요한지 한눈에 안 들어온다.

### 2.3 모드 선택 화면 문제

- Marathon, Sprint, Daily Challenge 카드가 기능 설명 카드처럼 보인다.
- 각 모드의 감정 톤이 구분되지 않는다.
- 모드 진입이 `선택`과 `이 모드 시작`으로 이중화되어 있는데, 시각적 임팩트는 약하다.

### 2.4 플레이 화면 문제

- 보드 외 영역이 너무 관리형이다.
- 점수, 라인, 레벨, Hold, Next Queue가 모두 같은 종류의 입력폼처럼 보여 게임 리듬이 느껴지지 않는다.
- 목표, 접근성, 조작 버튼이 한 화면 안에서 같은 강도로 놓여 긴장감이 분산된다.
- 현재 플레이 순간의 집중감보다 `설정 패널` 느낌이 강하다.

### 2.5 결과/랭킹 화면 문제

- 결과 화면이 승패/성취의 순간처럼 보이지 않는다.
- 기록 카드와 버튼이 너무 얌전해서 `다시 도전` 충동이 약하다.
- 랭킹 화면도 `경쟁`보다 `리스트 조회`에 가깝다.
- 닉네임 등록 시트와 설정 시트도 너무 행정적이다.

## 3. 사례 조사 요약

아래 내용은 2026-03-11 기준 공식 사이트, 공식 게임 페이지, 공식 스토어 설명을 바탕으로 한 `기획 해석`이다.

### 3.1 공식/성공 사례에서 공통적으로 보이는 방향

#### 사례 A: 공식 Tetris 모바일

공식 Tetris는 `quick round`, `single player modes`, `daily challenges`, `XP and rewards`, `custom themes/backgrounds/avatars`를 전면에 둔다. 즉, 단순 블록 게임이 아니라 `짧은 플레이`, `매일 할 이유`, `꾸미기`, `프로필화`, `즉시성`을 서비스 가치로 제시한다.

기획 해석:
- 화면도 단순 정보 나열이 아니라 `지금 한 판`, `오늘의 챌린지`, `보상`, `개인화`가 먼저 보여야 한다.

#### 사례 B: Candy Crush Soda Saga

공식 스토어 설명은 `colourful soda`, `juicy explosions`, `events and tournaments`, `leaderboard`, `daily rewards`, `boosters`를 강조한다. 핵심은 `화려한 감각 자극`, `계속 열리는 이벤트`, `보상`, `랭킹`, `매일 로그인 이유`다.

기획 해석:
- 캐주얼 게임은 정답형 정보 구조보다 `보상이 터질 것 같은 화면`, `지금 참여할 이벤트`, `쌓이는 혜택`이 더 먼저 느껴져야 한다.

#### 사례 C: Royal Match

Dream Games 공식 페이지는 `character design`, `world design`, `power-ups and boosters`, `areas`, `events`를 매우 강하게 설명한다. 성공한 캐주얼 게임은 단순 퍼즐 규칙만이 아니라 `캐릭터`, `풍부한 세계`, `부스터`, `잠금 해제`, `지속 이벤트`로 플레이 바깥의 감정을 만든다.

기획 해석:
- 우리 서비스도 라이선스 이슈를 피하는 독자 IP 전제라면, 더욱 `자체 세계관`, `전용 시각 언어`, `마스코트 또는 테마 오브젝트`가 필요하다.

#### 사례 D: Subway Surfers

공식 스토어 설명은 `colorful and vivid HD graphics`, `crew`, `hoverboard`, `events & offers`, `new outfits`, `rewards`를 강조한다. 즉 흥행형 캐주얼 게임은 플레이 화면 밖에서도 `활기`, `시즌성`, `수집`, `새로움`을 계속 공급한다.

기획 해석:
- 화면은 정적 카드가 아니라 `이 서비스는 살아 있고, 이벤트가 돌고 있고, 다음 보상이 기다린다`는 느낌을 줘야 한다.

## 4. 설계자에게 요구하는 화면설계 원칙

### 4.1 최상위 원칙

- 원칙 1: 화면은 `정보 정리`보다 `플레이 충동`을 우선해야 한다.
- 원칙 2: 사용자는 첫 3초 안에 `재미`, `속도감`, `보상`, `경쟁`을 느껴야 한다.
- 원칙 3: 독자 IP 전제이므로 공식 테트리스 트레이드드레스를 모방하지 말고, 별도의 캐주얼 세계관을 만들어야 한다.
- 원칙 4: 전 화면은 `게임 서비스`처럼 보여야지 `폼 기반 앱`처럼 보여서는 안 된다.
- 원칙 5: 결과, 랭킹, 데일리, 업적은 텍스트 설명이 아니라 시각적 보상 오브젝트로 느껴져야 한다.

### 4.2 비주얼 방향 강제 요구

- 배경은 단색에 가까운 베이지 카드 구조를 기본값으로 두지 말 것
- 전용 컬러 시스템을 만들 것
- 게임 로고 또는 서비스 타이틀에는 장난기 있는 보조 그래픽을 붙일 것
- 전용 마스코트, 도우미 캐릭터, 혹은 감정 표현이 가능한 오브젝트군을 둘 것
- 보상, 배지, 랭킹, 챌린지에는 아이콘이 아니라 `수집 욕구를 만드는 오브젝트화`가 필요할 것
- 버튼은 단순 pill 버튼에서 끝내지 말고, 상태 변화와 강조 계층을 더 강하게 나눌 것
- 카드 시스템은 2~3종으로 차등화할 것
- 승리/기록 갱신/데일리 완료 상태는 평상시 카드와 완전히 다른 시각 계층을 가질 것

## 5. 화면별 강화 요구사항

### 5.1 랜딩 화면

- Hero 영역에 `즉시 플레이 욕구`를 만드는 강한 비주얼이 있어야 한다.
- 첫 화면에는 최소 하나의 메인 장면이 필요하다.
  - 예: 블록이 쌓이며 폭발하는 장면
  - 예: 캐릭터/마스코트가 오늘의 챌린지를 소개하는 장면
  - 예: 최근 최고 기록 갱신 연출을 상징하는 쇼케이스
- `바로 시작`은 현재보다 훨씬 강한 1등 CTA여야 한다.
- `Daily Challenge`, `최근 성과`, `오늘의 목표`는 서로 다른 카드 계층으로 보여야 한다.
- 최근 성과는 단순 텍스트가 아니라 `배지`, `불꽃`, `갱신 마크`, `상위 퍼센트` 같은 보상 언어로 표현해야 한다.
- 접근성/설정은 메인 흥분을 해치지 않는 서브 위치로 내려야 한다.

### 5.2 모드 선택 화면

- Marathon, Sprint, Daily Challenge는 시각 테마가 달라야 한다.
- 모드 카드는 설명 카드가 아니라 `진입하고 싶은 스테이지 포스터`처럼 보여야 한다.
- 각 카드에는 최소 아래 요소가 있어야 한다.
  - 모드별 대표 비주얼
  - 한 줄 감정 카피
  - 핵심 목표
  - 보상 또는 경쟁 포인트
- `선택`과 `이 모드 시작`의 관계를 단순화해야 한다.
- 선택 상태는 아주 강하게 드러나야 한다.

### 5.3 플레이 화면

- 플레이 화면은 가장 먼저 `보드`가 보여야 하고, 나머지는 보드를 돕는 구조여야 한다.
- 정보 패널은 입력 폼처럼 보이지 않게 다시 설계해야 한다.
- 점수/콤보/레벨은 상승 중이라는 감각이 있어야 한다.
- Goal, Accessibility 같은 정보는 항상 노출되는 정적 카드가 아니라 접거나 보조 영역으로 이동하는 것이 적절하다.
- 조작 버튼은 더 게임패드처럼 보여야 한다.
- Next Queue, Hold, Score는 같은 스타일이 아니라 각각 역할이 다르게 보여야 한다.
- 배경과 보드 주변에는 플레이 긴장감을 주는 라이트, 그리드, 입자, 속도감 장치가 필요하다.

### 5.4 결과 화면

- 결과 화면은 지금보다 훨씬 `성과 화면`처럼 보여야 한다.
- 최종 점수, 최고 기록 갱신, 랭킹 반영 상태 중 최소 하나는 hero 수준으로 크게 보여야 한다.
- `재도전`은 화면에서 시각적으로 압도적인 1등 CTA여야 한다.
- 결과 카드들은 동일한 박스 반복이 아니라 중요도에 따라 크기와 강조가 달라야 한다.
- 첫 3판 최고 기록 갱신 시에는 일반 결과와 다른 축하 연출 전용 상태를 반드시 둘 것
- 공유 버튼은 결과를 자랑하고 싶게 만들어야 한다.

### 5.5 랭킹 화면

- 랭킹 화면은 표가 아니라 `경쟁 무대`처럼 보여야 한다.
- 내 기록은 일반 항목과 분리해 더 강하게 강조해야 한다.
- 상위 3위는 카드형 시상대 처리나 메달 처리 등으로 차등 표현해야 한다.
- 랭킹 탭은 기능 탭이 아니라 모드 배지처럼 보여야 한다.
- 순위 변동, 상위 퍼센트, 오늘 내 최고 위치 같은 비교 언어를 강화해야 한다.

### 5.6 시트/모달 화면

- 설정, 데일리, 닉네임 등록 시트는 지금처럼 밋밋한 폼이 아니라 각각 목적이 느껴져야 한다.
- Daily 시트에는 `오늘의 보상`, `달성 이유`, `챌린지 비주얼`이 반드시 있어야 한다.
- 닉네임 등록은 단순 입력이 아니라 `공식 기록에 이름을 새긴다`는 감정을 줘야 한다.
- 설정 시트는 정보량을 줄이고 조작 만족감을 높여야 한다.

## 6. 구체 요구사항 정의

### 6.1 Must

- UI 전체의 아트 방향을 현재 베이지 카드형 구조에서 게임 서비스형 구조로 재정의할 것
- 독자 IP 시각 언어를 만들 것
- 랜딩 Hero, 모드 카드, 결과 Hero, 랭킹 Top 3는 전면 재설계할 것
- 플레이 화면의 정보 패널을 폼 느낌이 나지 않게 재설계할 것
- Daily/성과/랭킹/보상에 전용 아이콘과 상태 연출을 설계할 것
- 첫 3초 안에 `바로 시작`, `오늘의 챌린지`, `최근 성과` 중 적어도 2개가 강하게 읽혀야 할 것
- 모바일 기준에서 단순 가독성뿐 아니라 `흥분감 있는 게임 화면`이 유지되어야 할 것

### 6.2 Should

- 전용 마스코트 또는 감정 표현 오브젝트를 도입할 것
- 시즌/이벤트에 따라 바뀔 수 있는 배경 또는 테마 레이어를 고려할 것
- 랭킹/결과 화면에 시상대, 메달, 불꽃, 스탬프류 표현을 도입할 것
- 버튼 상태 변화, 카드 등장, 기록 갱신 시 최소한의 모션 언어를 설계할 것
- 공유 화면은 자랑 욕구를 자극하는 전용 비주얼 레이아웃을 가질 것

### 6.3 Must Not

- 화면 대부분을 동일 톤의 박스 나열로 처리하지 말 것
- 플레이 화면을 설정 패널처럼 보이게 만들지 말 것
- 랜딩 Hero 없이 카드 리스트형 구조로 마무리하지 말 것
- 결과 화면을 단순 점수 폼과 버튼 반복으로 구성하지 말 것
- 공식 테트리스 라이선스 없이 공식 테트리스 트레이드드레스를 모사하지 말 것

## 7. 설계자 전달용 액션 아이템

1. 현재 8개 화면을 전면 재분류해 `게임 서비스형 아트 방향`으로 다시 제안할 것
2. 랜딩, 모드 선택, 플레이, 결과, 랭킹에 대해 각 1안이 아니라 최소 2가지 톤 앤 무드 방향을 제시할 것
3. 독자 IP를 위한 마스코트/테마 오브젝트/배지 체계를 제안할 것
4. `데일리`, `최근 성과`, `랭킹`, `공유`를 보상 언어로 바꾸는 UI 규칙을 설계할 것
5. 모바일 시안에서도 평면 문서형이 아닌 게임형 밀도를 유지하는지 자가 점검할 것

## 8. 기획자 최종 결론

현재 화면은 기능 확인용으로는 이해되지만, 흥행을 목표로 하는 캐주얼 게임 서비스 화면으로는 부족하다. 설계자는 이제 `읽히는 화면`이 아니라 `누르고 싶고, 자랑하고 싶고, 다시 오고 싶게 만드는 화면`을 설계해야 한다. 이 문서의 목적은 취향 제안이 아니라, `흥행형 게임 서비스 수준으로 요구사항을 상향`하는 데 있다.
