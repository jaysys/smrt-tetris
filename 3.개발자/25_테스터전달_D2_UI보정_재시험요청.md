# D2 UI 보정 재시험 요청

## 1. 목적
- 본 문서는 `4.테스터/11_D2_UI강화_시험결과서.md`의 미충족 항목 보정 후 재시험 범위를 전달하기 위한 문서다.
- 재시험 대상은 `FE-018`, `FE-019`, `FE-020`, `FE-021`, `FE-023`, `FE-026`, `FE-027`, `FE-029`, `FE-030`이다.
- `11_D2_UI강화_시험결과서.md`의 `Partial Pass`는 당시 기준의 유효한 중간 결과로 유지하되, 본 문서는 그 이후 보정 반영 상태를 기준으로 다시 시험하도록 요청하는 최신 인계 문서다.

## 2. 재시험 대상
- `TC-001A`
  - `360x800`에서 랜딩 `start-button`, `mode-button`, `ranking-button` 3개가 first-fold 안에 동시에 노출되는지 확인
- `TC-016`
  - `390x844`, `360x800`에서 결과 화면 CTA 4개 `retry-button`, `result-ranking-button`, `share-button`, `save-record-button`가 first-fold 안에 노출되는지 확인
- `TC-016A`
  - 결과 CTA 4개가 모바일 세로에서 2x2 구조를 유지하는지 확인
- `TC-026B`
  - 설정 시트에 `sheet-ghost-toggle`, `sheet-guide-toggle`이 노출되고 클릭 가능한지 확인
- `TC-PLAY-VIEWPORT`
  - `360x800` 기본 확인뿐 아니라 창 높이 축소 조건에서도 플레이 화면 `game-canvas`가 한 화면 안에 모두 보이는지 확인
  - 최소 확인 기준: `360x720`, `360x680`에서 `game-canvas`가 first viewport 안에 존재하고 좌우 여백 차이가 크지 않은지 확인
- `TC-HOME-AUTO-START`
  - `/` 첫 진입 시 랜딩을 거치지 않고 `Marathon` 플레이 화면이 바로 시작되는지 확인
  - 플레이 화면에서 `Escape` 입력 시 랜딩 화면으로 재진입 가능한지 확인
- `TC-PLAY-MINIMAL-UI`
  - 하단 온스크린 버튼이 전부 제거되어 있는지 확인
  - 플레이 화면에는 `game-canvas` 외에 상시 정보가 노출되지 않는지 확인
  - `play-viewport` 안에 보이는 버튼이 0개인지 확인
- `TC-PLAY-GAME-FEEL`
  - 랜딩/랭킹/결과/모드 선택에는 공통 게임형 아이콘과 엠블럼이 적용돼 화면이 대시보드가 아니라 게임 서비스처럼 보이는지 확인
- `TC-PLAY-HIDDEN-KEYS`
  - 플레이 중 `Escape` 입력 시 랜딩으로 이동하는지 확인
  - 플레이 중 `Q` 입력 시 결과 화면으로 이동하는지 확인

## 3. 이번 재시험에서 제외하는 범위
- 홈 하단의 검색용 가이드/FAQ 문구는 사용자 화면에 보이지 않도록 정리됐다.
- 이는 `FE-024` 범위이며 메타데이터, JSON-LD, 서버 렌더링 비가시 텍스트 유지가 목적이므로 본 UI 재시험의 시각 판정 대상에서는 제외한다.

## 4. 반영 내용
- 랜딩 CTA 3개를 Hero 상단으로 재배치했다.
- `360x800`에서 랜딩 CTA를 3열 레이아웃으로 고정했다.
- 결과 CTA 4개를 요약 카드보다 앞에 배치했다.
- 모바일 세로에서 결과 CTA 4개를 2x2 grid로 고정했다.
- 설정 시트에 `sheet-ghost-toggle`, `sheet-guide-toggle`을 추가했다.
- `guideOverlayEnabled` 설정값을 도입해 가이드 오버레이 노출 여부를 실제 상태로 관리한다.
- 플레이 화면은 `visualViewport`와 실측 레이아웃 높이를 기준으로 자동 축소되도록 변경했다.
- 공개 홈 `/` 첫 진입 시 `Marathon` 플레이가 자동 시작되도록 변경했다.
- 랜딩 화면은 플레이 중 `Escape` 키로 다시 열 수 있게 정리했다.
- bootstrap/API 요청은 timeout 이후 fallback으로 전환되도록 보강했고, `pnpm test:e2e`는 웹 빌드 후 Playwright start 서버만 띄우도록 실행 흐름을 분리했다.
- 랜딩, 모드 선택, 랭킹, 결과, 플레이에 공통 게임형 아이콘, 모드 엠블럼, 보상 오브젝트 계층을 반영했다.
- 플레이 화면은 `game-canvas`만 남기고 나머지 HUD, 텍스트, 키 안내, Next, 상시 버튼을 모두 제거했다.
- 플레이 중 랜딩 복귀는 `Escape`, 종료는 `Q` 키 경로로만 제공되도록 정리했다.
- 작은 높이 구간에서도 보드가 과도하게 작아지지 않도록, 보드 폭은 가용 높이와 폭 기준 최대치로 다시 계산하도록 보정했다.
