# 기획자 화면설계개선의견 설계자회신

## 1. 목적
- 본 문서는 `1.기획서/02_기획자_화면설계_개선의견서.md`에 대한 설계자 회신서다.
- 목적은 기획자의 화면설계 상향 요구를 수용 여부별로 정리하고, 실제 설계 문서 반영 방침을 명확히 전달하는 것이다.

## 2. 총평 회신
- 기획자의 총평은 타당하다.
- 현재 화면은 구조 검증용으로는 의미가 있지만, 흥행형 캐주얼 게임 서비스 화면 톤으로는 부족하다.
- 특히 `즉시 플레이`, `짧은 재도전`, `보상 기대`, `경쟁 무드`, `데일리 재방문`이 텍스트로만 존재하고 시각 경험으로는 약하다는 지적을 수용한다.

## 3. 항목별 회신
### 3.1 게임 서비스형 톤 상향 요구
판단:
- `수용`

회신:
- 기존 UI화면설계서는 구조와 위계 중심으로 작성되어 있었고, 감정 톤과 서비스 밀도 지시가 부족했다.
- 따라서 화면설계 문서에 `게임 서비스형 아트 방향`, `카드 계층`, `보상 언어`, `모션 언어`를 추가 반영한다.

### 3.2 독자 IP 시각 언어 필요
판단:
- `수용`

회신:
- 라이선스 미확보 전제에서는 공식 테트리스 트레이드드레스를 피하면서도, 독자적인 시각 언어를 만들어야 한다.
- 이를 위해 마스코트 또는 감정 표현 오브젝트, 배지 체계, 반복 테마 패턴을 UI 규칙에 포함한다.

### 3.3 랜딩 Hero 강화 요구
판단:
- `수용`

회신:
- 랜딩은 `정보 카드 집합`이 아니라 `바로 시작을 누르게 만드는 Hero 화면`이어야 한다.
- Hero 장면, 성과 오브젝트, Daily 보조 카드의 위계를 다시 고정한다.

### 3.4 모드 선택의 포스터화 요구
판단:
- `수용`

회신:
- Marathon, Sprint, Daily Challenge는 설명 카드가 아니라 감정이 다른 진입 포스터처럼 보여야 한다.
- 모드별 대표 비주얼, 감정 카피, 경쟁/보상 포인트를 필수 요소로 추가한다.

### 3.5 플레이 화면의 관리 패널 느낌 제거 요구
판단:
- `수용`

회신:
- 플레이 화면은 보드 우선 화면이어야 하며, 정보 패널은 폼 느낌이 나면 안 된다.
- 점수/콤보/레벨의 상승감, 게임패드형 조작부, 역할별로 다른 패널 언어를 반영한다.

### 3.6 결과/랭킹 화면의 성과감 강화 요구
판단:
- `수용`

회신:
- 결과는 성과 화면, 랭킹은 경쟁 무대로 읽혀야 한다는 지적은 타당하다.
- 결과 Hero, Top 3 시상대, 보상 배지, 공유 자랑 언어를 설계 기준으로 포함한다.

### 3.7 최소 2가지 톤앤무드 방향 제시 요구
판단:
- `수용`

회신:
- 개발자가 해석 가능한 수준으로 톤앤무드 방향을 문서에 직접 넣는 것이 맞다.
- 따라서 UI화면설계서에 권장 톤앤무드 2안을 추가한다.

## 4. 반영 결과
- `23_UI화면설계서.md`에 아래 항목을 보강했다.
  - 게임 서비스형 아트 방향
  - 독자 IP 시각 언어 기준
  - 톤앤무드 방향 2안
  - 카드 계층 규칙
  - 모션 언어 최소 기준
  - 랜딩/모드/플레이/결과/랭킹/Daily의 감정 설계 강화
- `24_개발자전달_UI화면설계_요약.md`에도 개발자가 바로 반영할 비주얼 방향 변경을 추가했다.

## 5. 설계자 최종 의견
- 기획자 의견은 대체로 전면 수용한다.
- 이번 보강으로 UI 설계 기준은 `구조 정리형 문서`에서 `게임 서비스형 화면 지시서` 수준으로 상향되었다.
- 이후 개발자는 단순 반응형 정리나 카드 배치 수정으로 대응하면 안 되고, 화면의 감정 톤과 보상 언어까지 함께 구현해야 한다.

## 6. 한 줄 회신
- 기획자의 화면설계 상향 요구는 수용하며, 화면설계 기준을 `정보 정리형 UI`에서 `흥행형 게임 서비스 UI`로 올리는 방향으로 문서를 보강해 반영했다.
