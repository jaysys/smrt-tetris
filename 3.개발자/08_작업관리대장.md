# 작업관리대장

## 1. 목적
본 문서는 개발 작업의 담당, 상태, 우선순위, 선행조건, 완료 기준을 관리하는 기준 문서다.

## 2. 운영 원칙
- 작업 상태가 바뀌면 같은 날 갱신한다.
- `DONE` 처리는 구현 완료가 아니라 QA 확인 및 문서 반영 완료까지 포함한다.
- 선행조건이 있는 작업은 blocker 여부를 함께 기록한다.
- 사용자에게 완료 상태 확인이나 다음 작업 진행 여부를 요청하기 전에는 이 문서를 먼저 최신 상태로 갱신한다.
- 사용자 보고 시 현재 작업, 완료 작업, blocker가 이 문서에 즉시 보이도록 유지한다.
- 문서 내 `갱신자` 표기는 항상 `Jaeho`로 유지한다.

## 3. 상태 기준
| 상태 | 의미 |
|---|---|
| `TODO` | 미착수 |
| `IN_PROGRESS` | 진행 중 |
| `BLOCKED` | 선행조건 또는 외부 결정 대기 |
| `DEV_DONE` | 구현 완료, 자체 검증 완료 |
| `QA_READY` | QA 가능 상태 |
| `QA_FAIL` | QA 실패 |
| `DONE` | QA 통과 및 문서 현행화 완료 |
