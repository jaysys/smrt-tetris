# 시각에셋 아이콘 배경 컬러가이드 검토의견서

## 1. 목적
- 본 문서는 `2.설계문서/27_개발자전달_시각에셋_아이콘_배경_컬러가이드.md`를 검토한 뒤, 수용 여부와 개발 반영 기준을 정리하기 위한 의견서다.

## 2. 검토 결론
- 방향은 타당하다.
- 구현 가능하다.
- 다만 본 문서는 `즉시 UI 코드 수정 지시서`라기보다 `에셋 소싱 + 색상 시스템 + 라이선스 운영 기준` 문서로 보는 것이 맞다.

내 의견은 다음과 같다.
- 현재 공개 웹 UI 강화 방향과 충돌하지 않는다.
- 오히려 `FE-014`, `FE-015`를 진행하기 전에 이 가이드를 기준으로 시각 언어를 고정하는 것이 맞다.
- 다만 실제 적용 전에 `원본 에셋 반입 절차`, `출처 표기 위치`, `프로젝트 내부 자산 폴더 구조`를 먼저 정해야 한다.

## 3. 항목별 의견
### 3.1 라이선스 원칙
- 의견: 전면 수용
- 판단:
  - 공식 게임의 로고, 캐릭터, UI 스크린샷을 직접 사용하지 않는다는 원칙은 반드시 유지해야 한다.
  - `톤`, `구성`, `색감 방향`, `보상 언어`만 참조한다는 기준도 타당하다.

### 3.2 에셋 소스 분류
- 의견: 수용
- 판단:
  - Utility icon과 Reward/Game icon을 분리한 계층 구조가 실무적으로 맞다.
  - Heroicons/Material Symbols와 Game-icons/Kenney를 같은 화면에서 같은 역할로 혼용하지 말라는 기준도 적절하다.

### 3.3 배경 가이드
- 의견: 수용
- 판단:
  - 랜딩, 플레이, 결과/랭킹의 배경 강도를 다르게 가져가라는 원칙은 현재 UI 강화 방향과 일치한다.
  - 특히 플레이 화면 배경 강도를 가장 약하게 둬 보드 가독성을 지키라는 기준은 유지해야 한다.

### 3.4 컬러 시스템
- 의견: 수용하되 프로젝트 토큰으로 재정의 필요
- 판단:
  - 제안된 팔레트 방향은 캐주얼 게임 서비스 톤에 적합하다.
  - 다만 현재 CSS 변수 체계와 연결되지 않았으므로, 실제 반영 시에는 프로젝트 토큰으로 재정의하고 단계적으로 교체해야 한다.

### 3.5 버튼 스타일 규칙
- 의견: 수용
- 판단:
  - 시작, 재도전, Daily, 랭킹 CTA를 서로 다른 색 체계로 분리하는 것은 좋다.
  - 다만 색만 바꾸는 수준이 아니라 상단 하이라이트, 하단 그림자, press 상태까지 같이 구현해야 요청서 의도에 맞다.

## 4. 현재 코드 기준 판정
| 구분 | 판정 | 의견 |
|---|---|---|
| 라이선스 회피 원칙 | 충족 | 공식 자산 직접 사용 없음 |
| 프로젝트 공통 컬러 토큰 | 미충족 | 현재는 기존 warm palette 중심 |
| CTA별 색상 분리 | 부분 충족 | 시작/기본 버튼 구분은 있으나 설계 수준 분리는 아님 |
| Reward icon 체계 | 미충족 | 실제 외부 아이콘 자산 미반입 |
| 배경 패턴 계층화 | 부분 충족 | gradient와 blob은 있으나 외부 배경 패턴 체계는 없음 |
| 크레딧/출처 표기 구조 | 미충족 | CC BY 계열 대응 구조 없음 |
| 자산 원본 보관 체계 | 미충족 | `assets-source` 또는 license inventory 부재 |

## 5. 개발 영향
- 현재 진행 중인 `FE-014`, `FE-015`에 직접 영향이 있다.
- 특히 아래 항목은 선행 또는 병행으로 등록해야 한다.
  - 프로젝트 컬러 토큰 재정의
  - CTA 스타일 공통 컴포넌트화
  - 시각 에셋 소스 폴더와 출처 기록 파일 생성
  - CC BY 대응용 credits/about 노출 위치 확정

## 6. 권고 조치
### 6.1 바로 등록할 작업
1. 프로젝트 공통 컬러 토큰을 캐주얼 UI 가이드 기준으로 재정의
2. CTA 버튼 스타일을 `start`, `retry`, `daily`, `rank` 4계열로 분리
3. `assets-source` 보관 정책과 라이선스 기록 파일 구조 정의
4. `credits/about` 노출 위치를 공개 웹 또는 별도 문서로 확정

### 6.2 후속 등록 작업
1. Reward/Game icon 세트 반입
2. 랜딩/결과/랭킹용 배경 패턴 자산 반입
3. 플레이 조작부를 게임패드형 시각으로 재구성
4. Top 3 시상대용 아이콘/배지 세트 적용

## 7. 특별히 주의할 점
- `Game-icons.net`, `Hero Patterns`, `BGJar` 일부는 크레딧이 필요하므로, 단순 다운로드 후 사용하면 안 된다.
- 외부 에셋을 코드에 반입하면 반드시 원본 출처 URL과 라이선스를 함께 기록해야 한다.
- 네트워크 접근이 필요한 실제 다운로드 단계는 별도 작업으로 분리하는 편이 안전하다.

## 8. 최종 의견
- 이 가이드는 수용하는 것이 맞다.
- 다만 디자인 참고 문서가 아니라 `자산/색상 운영 기준`이므로, 바로 CSS만 수정하는 방식으로 처리하면 누락이 생긴다.
- 따라서 `FE-014`, `FE-015`에 들어가기 전 최소한 `컬러 토큰`, `에셋 출처 기록`, `크레딧 처리 기준`을 먼저 고정하는 것이 타당하다.
