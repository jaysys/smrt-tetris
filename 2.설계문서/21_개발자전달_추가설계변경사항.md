# 개발자전달 추가설계변경사항

## 1. 목적
- 본 문서는 개발자에게 마지막으로 전달한 `18_개발자전달_설계변경사항.md` 이후 추가된 설계 변경만 정리한 차분 문서다.
- 이번 변경은 주로 `게임 화면 안에서 보이는 블록 동작 규칙`을 외부 참조 기준까지 포함해 명시적으로 고정한 것이다.

## 2. 기준점
- 이전 개발자 전달 문서: `18_개발자전달_설계변경사항.md`
- 이번 차분 기준일: `2026-03-11`

## 3. 이번 추가 변경의 핵심
- 기존 문서는 모바일, 분석, 접근성, 결과 연출, 광고 보호, 랭킹 규칙 중심 변경을 전달했다.
- 이번 추가 변경은 그 위에 더해 `코어 게임 엔진 규칙 자체`를 외부 참조 기준과 함께 명확히 고정한 것이다.
- 따라서 이번 문서는 UI/운영 정책 보강이 아니라 `엔진 구현 해석 금지선 설정`에 가깝다.

## 4. 개발팀 즉시 확인사항
### 4.1 블록 동작 규칙의 외부 참조 기준이 명시되었다
- `19_게임룰_외부참조기준.md`가 새로 추가되었다.
- 게임 화면 안 블록의 생성, 이동, 회전, Hold, Ghost 규칙은 아래 기준으로 해석한다.
  - 공식 제품 동작 확인: Tetris 공식 제품 페이지
  - 정량 메커닉 확인: TetrisWiki의 Guideline/SRS 계열 문서

개발 영향
- 이제 코어 블록 동작 규칙을 내부 추정이나 관성 구현으로 해석하면 안 된다.
- 공개 공식 사이트에 없는 정량 표는 TetrisWiki 기반 Guideline/SRS 표를 따르는 것으로 고정되었다.

### 4.2 블록 생성 규칙이 `7-bag`로 명시 고정되었다
- `12_게임규칙_수치테이블.md`에서 블록 생성 규칙을 외부 참조 기반 항목으로 다시 고정했다.
- 임의 랜덤 추출이 아니라 `7종 블록 1세트 셔플 후 순차 소비`를 사용한다.

개발 영향
- 엔진의 piece generator는 일반 RNG 방식으로 구현하면 안 된다.
- 테스트는 한 bag 안에서 7종이 중복 없이 1회씩만 나오는지 검증해야 한다.

### 4.3 회전 규칙이 SRS 기준으로 명시 고정되었다
- `01_기능설계서.md`와 `12_게임규칙_수치테이블.md`에서 회전 규칙을 `SRS(Super Rotation System)`로 명시했다.
- `J/L/S/T/Z` 공통 벽차기 표, `I` 전용 벽차기 표, `O` 예외 처리가 문서에 직접 들어갔다.
- 회전 실패 시 위치와 회전 상태 유지 규칙도 함께 고정되었다.

개발 영향
- 더 이상 `기본 회전만 구현`하거나 `간이 벽차기`로 대체하면 안 된다.
- 회전 테스트는 문서 표의 순서대로 5개 후보를 검사해야 한다.
- 보드 좌표계가 `y 아래 증가`라면 SRS 표 적용 시 `y` 부호 반전 처리가 필요하다.

### 4.4 Hold 규칙이 세부 수준까지 고정되었다
- Hold 초기 상태는 `비어 있음`으로 고정되었다.
- Hold는 블록 잠금 전 `1턴당 1회`만 허용한다.
- Hold로 다시 꺼낸 블록은 `현재 방향 유지`가 아니라 `스폰 방향으로 재정렬`한다.

개발 영향
- Hold 재호출 동작을 현재 회전 상태 유지로 구현하면 설계 위반이다.
- Hold 상태 전이는 별도 단위 테스트로 고정 확인해야 한다.

### 4.5 Ghost 규칙이 기본 제공 기준으로 고정되었다
- Ghost Piece는 기본값 `ON`이다.
- Ghost는 현재 블록의 즉시 착지 위치를 보여주는 시각 보조 요소다.

개발 영향
- Ghost를 단순 장식이 아니라 실제 현재 낙하 블록의 착지 예측값으로 계산해야 한다.
- Ghost는 충돌 판정의 실체가 아니라 표시 전용이어야 한다.

### 4.6 스폰 방향과 초기 좌표 기준이 더 명확해졌다
- `12_게임규칙_수치테이블.md`에 블록별 초기 상태, 스폰 오프셋, 회전 상태별 점유 셀 좌표가 정리되었다.
- 스폰 실패 판정 예시도 문서에 추가되었다.

개발 영향
- 블록 정의, 회전 상태 전이, 충돌 판정을 하나의 하드코딩 규칙 집합으로 맞출 수 있다.
- 프런트엔드 렌더링과 엔진 내부 좌표계가 다른 경우 매핑 규칙을 먼저 고정해야 한다.

### 4.7 기능설계서에서 외부 참조 문서 연결이 추가되었다
- `01_기능설계서.md`에 `19_게임룰_외부참조기준.md` 참조 문장이 들어갔다.
- 기능설계서만 읽어도 외부 참조 기준과 정량 기준 문서로 이동할 수 있게 바뀌었다.

개발 영향
- 개발자는 이제 `01 -> 19 -> 12` 순서로 규칙 근거를 확인할 수 있다.
- 구현 중 모호한 부분이 나오면 자체 룰을 만들지 말고 이 3문서 체계 안에서 해석해야 한다.

### 4.8 개발자 전용 엔진 구현 지시서가 추가되었다
- `20_개발자전달_게임엔진구현지시서.md`가 새로 추가되었다.
- 이 문서에는 다음이 정리돼 있다.
  - 반드시 고정할 엔진 룰
  - 구현 시 금지 해석
  - 권장 구현 순서
  - 최소 단위 테스트 요구

개발 영향
- 엔진 개발 착수 시 가장 먼저 참고할 전달 문서가 생겼다.
- 구현 순서와 테스트 우선순위를 설계 기준으로 바로 연결할 수 있다.

## 6. 개발팀 액션 요청
1. 현재 엔진 구현 또는 구현 예정 코드가 `7-bag`, `SRS`, `Hold 재정렬`, `Ghost 착지 예측`을 만족하는지 먼저 점검한다.
2. 블록 회전/이동 로직은 `12_게임규칙_수치테이블.md`의 표를 그대로 코드 기준값으로 옮긴다.
3. `J/L/S/T/Z`, `I`, `O`를 같은 회전 처리기로 단순화하지 않는다.
4. Hold와 Ghost는 후순위 UI 장식이 아니라 코어 엔진 규칙으로 간주하고 테스트를 작성한다.
5. 구현 중 문서와 다른 해석이 필요하면 임의 결정하지 말고 설계 검토 요청으로 올린다.

## 7. 최종 요약
- `18_개발자전달_설계변경사항.md` 이후의 가장 큰 추가 변경은 `코어 게임 규칙의 외부 참조 기준 명시`다.
- 이제 테스트 게임의 화면 내 블록 동작은 `내부 추정 규칙`이 아니라 `외부 참조가 있는 고정 규칙`으로 해석해야 한다.
- 개발팀은 이번 차분을 엔진 구현 기준 갱신으로 받아들여야 한다.
