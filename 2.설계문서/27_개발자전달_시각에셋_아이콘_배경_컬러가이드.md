# 개발자전달 시각에셋 아이콘 배경 컬러가이드

## 1. 목적
- 본 문서는 개발자가 공개된 에셋 사이트를 활용해 `캐주얼 게임 서비스처럼 보이는` 아이콘, 배경, 버튼, 색상 체계를 구현할 수 있도록 만드는 구체 가이드다.
- 목적은 현재의 밋밋한 화면을 `게임 서비스형 화면`으로 끌어올리되, 라이선스와 IP 리스크를 피하는 것이다.

## 2. 가장 중요한 원칙
- `공개된 사이트에 보인다`는 이유만으로 가져다 쓰면 안 된다.
- 반드시 `명시적 라이선스`가 있는 에셋만 사용한다.
- 공식 Tetris, Candy Crush, Royal Match, Subway Surfers의 로고, 스크린샷, 캐릭터, UI 이미지를 직접 가져와 쓰지 않는다.
- 공식 게임 사례에서는 `톤`, `구성`, `색감 방향`, `보상 언어`, `서비스 밀도`만 참조한다.
- 실제 아이콘/패턴/버튼 프레임은 별도 공개 라이선스 에셋 사이트에서 가져온다.

## 3. 참조한 캐주얼 게임 사례와 설계 해석
### 3.1 공식 Tetris
- 출처
  - https://tetris.com/products/video-game/tetris
- 확인 포인트
  - `Quick Play`, `Marathon`, `Daily Challenges`, `XP and rewards`, `custom themes`, `avatars`
- 설계 해석
  - 플레이 유도는 빠르게
  - Daily와 보상은 전면에
  - 개인화는 보조 루프로
  - 화면에는 `짧은 한 판`과 `다음 목표`가 함께 보여야 한다

### 3.2 Royal Match
- 출처
  - https://www.dreamgames.com/games/royal-match
- 확인 포인트
  - `character design`, `world design`, `power-ups`, `boosters`, `events`
- 설계 해석
  - 캐릭터 또는 감정 표현 오브젝트가 필요하다
  - Top 3, 배지, 이벤트는 단순 리스트가 아니라 보상 무대처럼 보여야 한다
  - 배경과 아이템은 둥글고 광택감 있는 형태가 유리하다

### 3.3 Candy Crush Soda Saga / Candy Crush
- 출처
  - https://www.king.com/fr/game/candycrush
  - https://play.google.com/store/apps/details?id=com.king.candycrushsodasaga
- 확인 포인트
  - `sweet`, `fizzy`, `events`, `weekly contest`, `leaderboard`, `prizes`
- 설계 해석
  - 데일리/성과/랭킹은 정보가 아니라 `달콤한 보상`처럼 보여야 한다
  - 카드와 버튼에는 밝은 하이라이트와 둥근 볼륨감이 필요하다
  - CTA는 기능 버튼보다 `누르고 싶은 사탕형 버튼`이 더 적합하다

### 3.4 Subway Surfers
- 출처
  - https://play.google.com/store/apps/details?id=com.kiloo.subwaysurf
- 확인 포인트
  - `colorful and vivid HD graphics`, `hoverboard`, `events & offers`, `new outfits`, `rewards`
- 설계 해석
  - 화면은 정적 카드보다 활기와 속도감이 느껴져야 한다
  - 배경은 살아 있는 월드처럼 느껴져야 하고, 버튼도 에너지 있는 컬러가 필요하다

### 4.3 이 프로젝트의 우선 사용 순서
1. `OpenGameArt.org`
   - 블록, 타일, 게임형 오브젝트, 배경 타일, 캐주얼 소품의 1차 소스
   - 단, `CC0` 자산을 최우선으로 고른다.
2. `Kenney`
   - 버튼 프레임, 패널, 박스 스킨, 입력 프롬프트, 게임형 오브젝트 보강
3. `Game-icons.net`
   - 보상/랭킹/챌린지/메달/배지
4. `Unsplash`, `Pexels`, `Pixabay`
   - Hero 배경, 추상 배경, 월드 무드 보강
5. `Heroicons`, `Material Symbols`, `Font Awesome Free`
   - 설정, 공유, 닫기, 뒤로가기 같은 시스템 아이콘
6. `Flaticon`
   - 다른 소스로 충분하지 않을 때만, 출처 표기 계획이 있을 경우에 한해 사용

### 4.4 OpenGameArt 자산 선택 우선순위
OpenGameArt를 사용할 경우에는 사이트 전체를 한 번에 승인하지 말고, 아래 우선순위로 선별한다.

1. `CC0`
   - 1순위
   - 출처 표기 부담이 가장 적고, 상업적 사용과 수정이 가장 자유롭다.
2. `OGA-BY`
   - 2순위
   - OpenGameArt 표준 출처 표기 조건을 감당할 수 있을 때 사용한다.
3. `CC-BY`
   - 3순위
   - 일반 출처 표기 조건을 감당할 수 있을 때 사용한다.
4. `CC-BY-SA`, `GPL`, `LGPL`
   - 원칙적으로 비권장
   - 꼭 필요한 자산이 아닐 경우 피한다.

정리:
- OpenGameArt에서 `1순위로 추천하는 사용 가능 자산`은 `CC0`다.
- 다음 우선순위는 `OGA-BY`, 그다음은 `CC-BY`다.
- 개발자는 다운로드 전에 반드시 개별 자산 라이선스를 확인해야 한다.
- 본 프로젝트에서는 다른 무료 에셋 사이트보다도 `OpenGameArt`를 먼저 검토하는 것을 기본 원칙으로 한다.

## 6. 아이콘 체계 정의
### 6.1 아이콘 계층
- Utility Icon
  - 설정, 닫기, 뒤로가기, 공유, 음소거, 토글
  - 소스: Heroicons, Material Symbols, Font Awesome Free
- Game Reward Icon
  - 트로피, 메달, 불꽃, 별, 스탬프, 왕관, 선물
  - 소스: Game-icons.net 또는 Kenney UI Pack
- World Object Icon
  - 마스코트 소품, 챌린지 토큰, 랭킹 오브젝트, 데일리 티켓
  - 소스: Kenney UI Pack, Pixabay 벡터, OpenGameArt 개별 승인 자산
- Input Prompt Icon
  - 터치, 키보드, 게임패드, 제스처
  - 소스: Kenney Input Prompts

### 6.2 화면별 아이콘 정의
| 화면/기능 | 필요한 아이콘 성격 | 추천 소스 | 추천 키워드 |
|---|---|---|---|
| 랜딩 `바로 시작` | 에너지/플레이/스파크 | Game-icons.net, Kenney | `play-button`, `sparkles`, `electric` |
| 랜딩 `오늘의 챌린지` | 캘린더+보상 | Game-icons.net, Kenney, Font Awesome | `calendar`, `gift`, `quest`, `ticket` |

| 모드 Marathon | 생존/성장 | Game-icons.net, OpenGameArt | `infinity`, `tower`, `stairs-goal` |
| 모드 Sprint | 속도/타이머 | Game-icons.net, Font Awesome | `stopwatch`, `rocket`, `speedometer` |
| 모드 Daily | 오늘의 미션/보상 | Game-icons.net, Kenney | `calendar-star`, `gift`, `scroll-unfurled` |
| 결과 PB | 기록 갱신 | Game-icons.net, Kenney | `trophy-cup`, `star-medal`, `sparkler` |

### 6.3 실제 적용 규칙
- `설정`, `닫기`, `뒤로가기` 같은 시스템 기능은 Heroicons/Material Symbols를 우선한다.
- `랭킹`, `보상`, `챌린지`, `PB`, `공유 자랑`은 Game-icons 또는 Kenney 계열을 써서 게임 느낌을 만든다.
- 같은 화면 안에서 유틸리티 아이콘과 게임 보상 아이콘을 같은 스타일로 섞지 않는다.
- Game-icons 사용 시 크레딧 페이지 또는 about 화면에 출처를 남긴다.
- Flaticon은 무료 사용 조건이 복잡하므로, 같은 목적이면 Heroicons/Material Symbols/Font Awesome/Game-icons를 먼저 쓴다.

## 7. 배경 가이드
### 7.1 배경 방향 원칙
- 배경은 평면 단색 금지
- 보드 가독성은 유지
- 화면마다 다른 감정 톤 제공
- 서비스가 살아 있다는 느낌 제공

### 7.2 배경 생성 소스와 사용법
#### 랜딩
- 추천 소스
  - Unsplash `cyberpunk`, `arcade`, `glow`, `abstract dark`
  - Pexels `neon background`, `gradient abstract`, `arcade lights`
  - Pixabay `arcade background`, `abstract neon vector`
  - BGJar `Glowing Stars`, `Colored Shapes`, `Simple Shiny`
  - Hero Patterns의 반복 패턴
- 적용법
  - 사진 배경은 그대로 쓰지 말고 20~35% dark overlay와 색 보정을 건다.
  - SVG 배경을 직접 깔고, 위에 큰 블러 오브젝트 2~3개를 오버레이한다.
  - Hero 뒤에 약한 입자 또는 라이트를 넣는다.

#### 플레이 화면
- 추천 소스
  - Pixabay의 추상 벡터 격자/광선 배경
  - OpenGameArt의 CC0 또는 OGA-BY 배경 타일
  - BGJar `Circuit Board`, `Rect Light`, `Dash Wave`
  - Hero Patterns의 얇은 격자 패턴
- 적용법
  - 매우 낮은 대비의 그리드/회로형 패턴을 배경 뒤에 둔다.
  - 보드 주변에만 청색/청록 계열 라이트를 약하게 배치한다.

#### 결과 / 랭킹
- 추천 소스
  - Unsplash / Pexels의 보케형 라이트 배경
  - Pixabay의 스탬프/메달형 벡터 배경
  - BGJar `Shiny Overlay`, `Concentric Circle`, `Polygon Luminary`
  - Hero Patterns 반복 패턴
- 적용법
  - 결과는 중앙 Hero 카드 뒤에 방사형 광원 또는 종이 스탬프 느낌을 준다.
  - 랭킹은 시상대/트로피 뒤에 금색 또는 보라색 광원 레이어를 둔다.

### 7.3 배경 강도 규칙
- 보드 화면: 가장 약함
- 랜딩: 중간
- 결과/랭킹: 가장 강함
- Daily/설정/닉네임 시트: 중간 이하

## 8. 컬러 시스템 제안
### 8.1 사례 기반 색상 방향
- Tetris 계열: 짙은 네이비 + 네온 청록/자홍
- Royal Match 계열: 로열 블루 + 금색 보상
- Candy Crush 계열: 핑크/보라/오렌지의 광택 CTA
- Subway Surfers 계열: 하늘색/노랑/주황의 활기

- 아래 색상은 공식 게임 색상 추출본이 아니라, 위 사례의 시각 방향을 참고해 만든 프로젝트용 제안값이다.

| `--success` | `#64E58E` | 성공 상태 |
| `--danger` | `#FF6B6B` | 실패/경고 |
| `--text-main` | `#F6F8FF` | 기본 텍스트 |
| `--text-sub` | `#B8C4F2` | 보조 텍스트 |

### 8.3 버튼 컬러 규칙
- `바로 시작`
  - 주황 -> 노랑 그라데이션
  - 캐주얼 게임에서 가장 잘 읽히는 warm CTA
- `재도전`
  - 청록 -> 블루 그라데이션
  - 빠른 반복 플레이에 적합
- `도전 시작`
  - 연두 -> 초록 그라데이션
  - 보상/미션 감각 강화
- `랭킹`
  - 보라 -> 블루 그라데이션
  - 경쟁과 프리미엄 느낌

### 8.4 버튼 스타일 규칙
- 둥근 직사각형 + 상단 하이라이트 + 하단 그림자
- 기본 높이 모바일 `56px`
- 상단 반사광 1줄 필수
- press 상태에서는 `translateY(2px)`와 그림자 축소
- disabled 상태는 단순 gray가 아니라 `desaturated blue-gray`로 처리

### 8.5 캐주얼 게임형 버튼 프레임 소스
- Kenney UI Pack의 라운드 버튼 프레임을 기본 추천한다.
- Sci-Fi 계열은 `플레이/랭킹`, Adventure 계열은 `Daily/보상`, 기본 UI Pack은 `설정/보조 버튼`에 적합하다.
- Flaticon이나 Font Awesome 아이콘은 버튼 프레임이 아니라 내부 심볼에만 사용한다.

### 8.6 박스와 패널 스킨 규칙
- 게임 화면, 결과 화면, 랭킹 화면에 보이는 모든 박스와 버튼은 `Kenney UI Pack` 계열 프레임을 적용한다.
- 기본 CSS 사각형 박스를 그대로 남기지 않는다.
- HUD 박스, 키보드 안내 박스, 일시정지 버튼은 `Sci-Fi` 계열을 우선한다.
- 결과 Hero, 재도전 버튼, 보상 카드, 랭킹 Top 3는 `Adventure` 또는 기본 UI Pack 계열을 우선한다.
- 하나의 화면 안에서는 프레임 계열을 1개 주계열 + 1개 보조계열 정도로 제한한다.

## 9. 화면별 구현 지시
### 9.1 랜딩
- BGJar 배경 + Hero 오브젝트 + warm CTA 사용
- `바로 시작` 버튼은 무조건 `--cta-start -> --cta-end`
- 최근 성과 카드에는 `--reward-gold`, `--reward-pink` 강조 배지 사용

### 9.2 모드 선택
- Marathon: 네온 청색 계열
- Sprint: 오렌지/레드 속도 계열
- Daily: 초록/금색 보상 계열
- 카드마다 테마를 달리하고, 같은 프레임 복붙 금지

### 9.3 플레이
- 배경은 `--bg-deep -> --bg-mid` 계열
- 버튼은 불투명한 다크 패널이 아니라 컬러가 살아 있는 게임패드형 버튼 사용
- Ghost Piece는 실제 블록과 같은 계열 색상 + `opacity 0.12 ~ 0.18` 수준의 매우 연한 톤 사용
- Hold / Next / Score 패널에는 Kenney 프레임 또는 그와 유사한 둥근 패널 언어를 적용한다.
- HUD / 키 안내 / 일시정지 버튼에는 `Kenney UI Pack - Sci-Fi` 프레임을 적용하고, 기본 CSS 사각 박스는 금지한다.

### 9.4 결과
- Hero 카드 뒤에 보상 광원 배치
- PB 갱신 시 `--reward-pink` + `--reward-gold` 혼합 강조
- `재도전` 버튼은 `--retry-start -> --retry-end`
- Hero 카드와 CTA 버튼은 `Kenney UI Pack` 또는 `Adventure` 프레임을 적용한다.

### 9.5 랭킹
- Top 3는 `금 / 보라 / 청색` 차등 강조
- 메달, 트로피, 월계수 계열 오브젝트 사용
- 일반 리스트와 시상대 영역을 시각적으로 분리

## 10. 가져오는 방법 가이드
### 10.1 아이콘
1. Heroicons, Material Symbols, Font Awesome Free에서 유틸 아이콘 선택
2. Game-icons, Kenney, 필요 시 OpenGameArt에서 게임 보상 아이콘 선택
3. SVG 원본 다운로드
4. 프로젝트 스타일에 맞게 색상 재정의
5. `assets-source` 보관 폴더에 원본과 라이선스 URL 기록

### 10.2 버튼/패널 프레임
1. Kenney UI Pack 또는 UI Pack Adventure/Sci-Fi에서 기본 프레임 선택
2. 필요한 요소만 추출
3. 색상을 프로젝트 토큰으로 재색상
4. 텍스트와 아이콘을 올려 합성

### 10.4 폰트
1. `Black Han Sans`, `Orbitron`, `Signika` 원본을 설치 또는 웹폰트로 로드
2. 타이틀/CTA, HUD 숫자, 본문/키 안내에 각각 분리 적용
3. 시스템 기본 폰트가 보이는 화면이 남아 있지 않게 점검

### 10.3 배경
1. Unsplash / Pexels / Pixabay / OpenGameArt / Hero Patterns / BGJar 중 화면 목적에 맞는 원본 선택
2. 사진 배경은 dark overlay와 blur를 적용하고, SVG 배경은 색상과 밀도를 조정
3. 그대로 쓰지 말고, 블러 오브젝트/그라데이션/노이즈를 추가
4. 보드 대비를 체크한 뒤 최종 적용

## 11. 라이선스 기록 규칙
- 각 에셋마다 아래 4가지를 기록한다.
  - 원본 URL
  - 라이선스
  - 수정 여부
  - 사용 화면
- `Game-icons`, `Hero Patterns`, `BGJar Free`는 출처 표시 필요 여부를 반드시 체크한다.
- `OpenGameArt`는 자산마다 라이선스가 다르므로 다운로드마다 별도 기록한다.
- `Flaticon` 무료 자산은 출처 표기 조건을 감당할 수 있을 때만 사용한다.
- `Font Awesome Free`는 아이콘, 폰트, 코드 라이선스가 다르므로 패키지 사용 방식을 함께 기록한다.
- 출시 전 about/credits 또는 별도 attribution 페이지를 준비한다.

## 12. 개발자 최종 요청
- 현재 밋밋한 화면을 개선하려면, 단순히 색 몇 개 바꾸는 수준으로는 부족하다.
- 아이콘 체계, 배경 생성 방식, 버튼 컬러 계층, 보상 오브젝트를 함께 바꿔야 한다.
- 구현은 아래 순서로 진행한다.
1. 랜딩 Hero와 `바로 시작` CTA 재설계
2. 결과 Hero와 `재도전` CTA 재설계
3. 랭킹 Top 3 보상 무대화
4. Daily/최근 성과 카드의 보상 언어 적용
5. 플레이 화면의 조작 버튼과 배경 강화

## 13. 최종 결론
- 개발자는 공식 게임 화면을 복제하면 안 된다.
- 대신 공식/성공 사례에서 `캐주얼 게임 서비스의 감정 구조`를 참조하고, 실제 에셋은 공개 라이선스 사이트에서 가져와 조합해야 한다.
- 본 문서는 그 조합 방법을 구체적으로 고정한 개발 가이드다.
