# 용어사전 및 가정사항

## 1. 용어사전
| 용어 | 정의 |
|---|---|
| 게스트 사용자 | 로그인 없이 토큰으로 식별되는 사용자 |
| 공식 기록 | 서버 검증을 통과해 랭킹 반영 가능한 기록 |
| 개인 최고 기록 | 사용자별 모드 최고 점수 또는 최고 시간 |
| Daily Challenge | 매일 갱신되는 운영형 챌린지 |
| 반복 미션 | 짧은 주기로 반복 가능한 행동 기반 목표 |
| 업적 | 장기 목표형 달성 항목 |
| Hold | 현재 블록을 보관하고 다음 블록과 교체하는 기능 |
| Ghost Piece | 현재 블록의 예상 착지 위치 미리보기 |
| Fraud Flag | 비정상 기록 또는 부정행위 의심 상태 |
| Config Version | 게임 규칙과 운영 설정의 버전 단위 |

## 2. 설계 가정사항
- 본 서비스는 MVP에서 계정 연동 없이 게스트 기반으로 시작한다.
- 결과 공유는 링크 중심으로 먼저 제공한다.
- 결제와 광고는 MVP 범위에 포함하지 않는다. 단, 배치 위치 정책은 문서상 정의한다.
- Daily Challenge는 하루 1개만 활성화한다.
- 운영 시간대 기준은 한국 시간 UTC+09:00을 기본으로 한다.

## 2.1 시간 기준 통일 원칙
- 시스템 내부 저장 시간은 UTC다.
- 사용자 표시와 운영 집계 경계는 KST(UTC+09:00)다.
- 문서 간 시간 기준이 충돌할 경우 `저장=UTC`, `표시/집계=KST` 원칙을 우선한다.

## 3. 정책 가정
- 법무 검토 전까지 외부 노출 서비스명은 가칭을 사용한다.
- 블록 명칭과 특정 시각 표현은 일반적 표현으로 제한한다.
- 닉네임 중복은 허용하되 금칙어는 차단한다.
- 과금 요소가 추가되더라도 공식 기록 경쟁력에 영향을 주면 안 된다.
- 광고 또는 수익화 UI는 코어 플레이 중간과 재도전 직전 강제 노출을 금지한다.

## 4. 범위 제외 항목
- 실시간 멀티플레이
- 친구/팔로우 소셜 그래프
- 시즌 패스 상점
- PWA 설치 플로우
- 오프라인 모드
- 리플레이 저장

## 5. 미확정 이슈
| 항목 | 설명 | 영향 |
|---|---|---|
| 라이선스 여부 | 정식 라이선스 확보 시 브랜딩 정책 변경 가능 | UI 명칭, 법무 문구 |
| 로그인 도입 시점 | MVP 이후 도입 가능 | 친구 기능, 복귀 채널 |
| 광고 적용 여부 | 릴리스 이후 수익화 검토 | 결과 화면 UX |
| 시즌 운영 시작일 | 초기 운영 전략 미확정 | 랭킹 초기화 정책 |
| 복귀 사용자 경험 방식 | 복귀 배너/복귀 미션/기록 회상/데일리 재진입 유도 중 선택 필요 | CRM, 리텐션 UX |

## 6. 문서 해석 원칙
- `Must`는 MVP 릴리스 전 반드시 충족해야 한다.
- `Should`는 설계 반영 후 일정에 따라 릴리스 범위를 결정한다.
- `Could`는 구조만 고려하고 구현은 후속 검토한다.
