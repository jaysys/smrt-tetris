# 화면흐름도 및 사용자시나리오

## 2. 메인 흐름
```text
SC-01 랜딩
  -> 바로 시작 -> SC-03 게임 화면
  -> 모드 선택 -> SC-02 모드 선택 -> SC-03 게임 화면
  -> 랭킹 -> SC-06 랭킹 화면

SC-03 게임 화면
  -> 일시정지 -> SC-04
  -> 게임 종료 -> SC-05 결과 화면

SC-05 결과 화면
  -> 재도전 -> SC-03
  -> 랭킹 -> SC-06
  -> 닉네임 등록 -> SC-07
  -> 공유 -> 공유 시트 또는 공유 페이지
```

## 3. 사용자 시나리오
## 3.1 시나리오 A: 첫 방문자가 1판 플레이하고 재도전
1. 사용자가 랜딩 화면에 진입한다.
2. `바로 시작`을 누른다.
3. 튜토리얼 힌트 오버레이가 뜨고 사용자는 스킵한다.
4. 게임 화면에서 첫 판을 플레이한다.
5. 게임 오버 후 결과 화면을 본다.
6. `재도전`을 눌러 즉시 2판째를 시작한다.

성공 기준
- 첫 진입부터 첫 판 시작까지 추가 인증이 없다.
- 결과 화면에서 재도전 버튼이 가장 먼저 보인다.

## 3.1A 시나리오 A-확장: 첫 3판 최고 기록 갱신 연출
1. 사용자가 첫 3판 이내에 이전 개인 최고 기록보다 더 좋은 기록을 만든다.
2. 결과 화면이 열린다.
3. 결과 카드에 일반 결과와 구분되는 강조 연출이 노출된다.
4. 사용자는 강조된 `재도전` 흐름을 통해 다음 판을 시작할 수 있다.

성공 기준
- 축하 카피 또는 배지가 노출된다.
- 결과 카드 강조가 일반 결과와 시각적으로 구분된다.
- 재도전 CTA가 연출과 분리되지 않고 바로 이어진다.

## 3.2 시나리오 B: 기록 등록과 랭킹 확인
1. 사용자가 Sprint 모드에서 기록을 달성한다.
2. 결과 화면에서 `공식 랭킹 등록`을 선택한다.
3. 닉네임 등록 모달이 열린다.
4. 닉네임 입력 후 저장한다.
5. 서버 검증 완료 후 랭킹 화면으로 이동한다.

성공 기준
- 닉네임 등록 후 동일 결과가 다시 제출되지 않는다.
- 랭킹에서 방금 등록한 기록을 확인할 수 있다.

## 3.3 시나리오 C: Daily Challenge 수행
1. 사용자가 랜딩 또는 모드 선택에서 Daily Challenge 배너를 누른다.
2. 오늘의 목표와 보상을 확인한다.
3. 챌린지를 시작한다.
4. 목표 달성 후 결과 화면에서 보상 수령 버튼이 활성화된다.

성공 기준
- 보상은 1회만 수령된다.
- 실패 후 재도전 가능 메시지가 노출된다.

## 3.4 시나리오 D: 운영자 챌린지 교체
1. 운영자가 백오피스 로그인한다.
2. 다음 날 Daily Challenge를 생성한다.
3. 미리보기로 시작/종료 시각과 보상 수치를 확인한다.
4. 저장 후 예약 상태를 확인한다.

성공 기준
- 저장된 챌린지가 자정 이후 자동 활성화된다.

## 4. 화면별 필수 UI 요소
## 4.1 SC-01 랜딩
- 메인 로고
- `바로 시작`
- `모드 선택`
- `랭킹`
- Daily Challenge 배너
- 공지 요약

## 4.2 SC-03 게임 화면
- 캔버스 보드
- 다음 블록
- Hold
- 점수, 라인, 레벨
- 개인 최고 기록 상태
- 모바일 터치 조작부

## 4.3 SC-05 결과 화면
- 최종 결과 카드
- 개인 최고 기록 여부
- 랭킹 반영 상태
- 재도전 CTA
- 공유 CTA
- 닉네임 등록 또는 기록 저장 CTA
- 첫 3판 최고 기록 갱신 시 축하 카피 또는 배지
- 첫 CTA 영역 비침범 규칙

## 5. 모바일 세로 UX 기준
- 보드와 HUD는 상하 배치로 구성한다.
- 좌우 이동, 회전, 드롭 버튼은 엄지 영역 기준 하단 고정 배치다.
- 결과 화면 버튼은 세로 스택이 아니라 2열 배치를 우선 적용한다.
- 기준 뷰포트는 `390x844`, `360x800` 두 가지를 모두 사용한다.

## 5.1 코어 루프 보호 규칙
- 플레이 도중 광고나 수익화 UI를 삽입하지 않는다.
- 첫 결과 화면의 첫 CTA 영역은 광고나 프로모션 요소가 침범하지 않는다.
- `재도전` 직전 강제 광고는 허용하지 않는다.

## 6. 화면 전이 예외
- 네트워크 불안정 시 결과 제출 실패해도 결과 화면은 유지하고 `재전송` 상태 배지를 노출한다.
- 랭킹 API 실패 시 랭킹 화면 대신 `일시적으로 랭킹을 불러오지 못했습니다` 안내를 띄운다.
- 설정 저장 실패 시 이전 설정으로 롤백하고 토스트를 띄운다.
