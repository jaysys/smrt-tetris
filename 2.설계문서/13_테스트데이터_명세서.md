# 테스트데이터 명세서

## 1. 목적
개발자와 제3자 테스터가 동일한 데이터 조건에서 테스트를 재현할 수 있도록 기본 계정, 시드 데이터, 초기화 절차를 정의한다.

## 2. 테스트 URL
| 환경 | URL |
|---|---|
| 로컬 웹 | `http://localhost:3000` |
| 로컬 API | `http://localhost:3001/v1` |
| 로컬 운영 | `http://localhost:3000/admin` |

## 3. 기본 계정
| 구분 | ID/닉네임 | 용도 |
|---|---|---|
| 게스트 A | 자동 발급 | 기본 플레이 |
| 게스트 B | 자동 발급 | 중복/재시도 테스트 |
| 운영자 | `admin_test` | 챌린지/공지/제재 테스트 |

운영자 초기 정보
- 아이디: `admin_test`
- 비밀번호: `ChangeMe123!`
- 최초 로그인 후 즉시 변경

## 4. 기본 시드 데이터
| 데이터 종류 | 기본값 |
|---|---|
| 모드 | `MARATHON`, `SPRINT`, `DAILY_CHALLENGE` |
| 오늘의 챌린지 | `3분 내 20라인 제거` |
| 반복 미션 | `오늘 3판 플레이`, `랭킹 1회 조회`, `개인 최고 기록 1회 갱신` |
| 업적 | 첫 플레이 완료, Sprint 첫 완주 등 5종 |
| 금칙어 | 욕설/광고/관리자 사칭 샘플 20개 이상 |

## 5. 랭킹 샘플 데이터
- Marathon: 41000점, 32000점, 18000점
- Sprint: 178250ms, 201300ms, 265100ms
- Daily Challenge: 25000점, 19000점, 12000점

## 6. DB 초기화 절차
```bash
pnpm db:reset
pnpm db:migrate
pnpm db:seed
```

## 7. 날짜 전환 테스트 데이터
- 서버 프로세스 시간은 UTC 유지
- 운영 기준 날짜는 KST 계산 사용
- 날짜 전환 테스트는 다음 3시점을 준비한다
  - KST 23:59:50
  - KST 00:00:00
  - KST 00:00:10

## 8. 네트워크 장애 테스트 데이터
- 결과 제출 직전 API 응답 지연 3초
- 결과 제출 요청 1회 실패 후 재시도
- 랭킹 조회 500 오류 1회
