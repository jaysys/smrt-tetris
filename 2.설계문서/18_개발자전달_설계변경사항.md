# 개발자전달 설계변경사항

## 1. 목적
- 본 문서는 최근 설계 문서 반영 사항 중 개발팀이 즉시 알아야 할 변경사항만 추려 전달하기 위한 문서다.
- 기준 문서는 `17_기획자검토의견_설계자회신.md` 이후 갱신된 `2.설계문서` 일체다.

## 2. 적용 기준일
- 기준일: `2026-03-11`

## 3. 변경 배경
- 기획자 검토 의견 수용에 따라 모바일 우선순위, 접근성 최소 범위, 핵심 분석 이벤트, 랭킹 동점 규칙, 초기 성취감 연출, 광고 보호 규칙, 복귀 루프 방향성이 설계 본문에 보강되었다.
- 이번 변경은 단순 문구 수정이 아니라 개발 우선순위와 테스트 기준에 직접 영향을 준다.

## 4. 개발팀 즉시 확인사항
### 4.1 개발 단계 우선순위 변경
- `06_MVP개발계획서.md` 기준으로 아래 항목이 앞당겨졌다.
  - Phase 1: 모바일 세로 기본 HUD/조작부 레이아웃, 핵심 퍼널 이벤트
  - Phase 2: 모바일 세로 레이아웃과 모드별 화면 검증
  - Phase 3: 접근성 최소 옵션
- 따라서 모바일 세로 기본 대응, 핵심 이벤트 수집, 접근성 최소 옵션은 더 이상 출시 직전 보강 항목이 아니다.

개발 영향
- D1 구현 시 모바일 세로 레이아웃을 임시가 아니라 완료 조건으로 봐야 한다.
- 분석 이벤트는 코어 루프 완료 후 붙이는 후순위 작업이 아니라 코어 루프와 함께 구현해야 한다.
- 접근성 최소 옵션은 결과 제출 이후 별도 단계가 아니라 기록/경쟁 루프 이전에 준비해야 한다.

### 4.2 Marathon 랭킹 동점 처리 규칙 확정
- `01_기능설계서.md`의 Marathon 랭킹 기준이 아래 순서로 고정되었다.
  - 1순위: 점수 내림차순
  - 2순위: 총 생존 시간 오름차순
  - 3순위: 총 입력 수 오름차순
  - 4순위: 먼저 달성한 기록 우선

개발 영향
- 랭킹 API, DB 조회 정렬, 테스트 데이터, QA 기대결과를 이 기준에 맞춰야 한다.
- 기존의 모호한 문구 해석을 더 이상 허용하지 않는다.

### 4.3 첫 3판 최고 기록 강화 연출 추가
- `01_기능설계서.md`의 결과 화면 규칙에 아래 항목이 추가되었다.
  - 첫 3판 이내 개인 최고 기록 갱신 시 일반 결과와 구분되는 강조 연출
  - 축하 카피 또는 배지
  - 결과 카드 시각 강조
  - `재도전` CTA와 인접한 위치에서 재도전 유도

개발 영향
- 결과 화면 컴포넌트는 단일 상태가 아니라 `초기 3판 PB 강조 상태`를 추가로 처리해야 한다.
- 이벤트 트래킹과 연계 시 이 상태를 식별 가능한 값으로 남기는 것을 권장한다.

### 4.4 핵심 분석 이벤트 최소 세트 확정
- `02_시스템설계서.md`, `06_MVP개발계획서.md`, `14_QA실행가이드.md` 기준으로 아래 이벤트가 최소 수집 세트로 고정되었다.
  - `landing_view`
  - `quick_start_click`
  - `game_start`
  - `game_finish`
  - `retry_click`

개발 영향
- 이벤트 명칭은 임의 변경하지 않는다.
- 프론트엔드와 백엔드는 이 다섯 이벤트를 기준 이름으로 맞춘다.
- 자동화와 운영 모니터링도 이 다섯 이벤트를 기준으로 본다.

### 4.5 광고/수익화 금지 구역 규칙 추가
- `01_기능설계서.md`, `10_운영배포체크리스트.md`, `11_용어사전_및_가정사항.md`에 아래 규칙이 고정되었다.
  - 플레이 도중 광고 금지
  - 첫 결과 화면의 첫 CTA 영역 침범 금지
  - `재도전` 직전 강제 광고 금지

개발 영향
- 결과 화면 레이아웃 설계 시 첫 CTA 영역은 프로모션 슬롯으로 사용하면 안 된다.
- 향후 광고 슬롯이나 프로모션 배너 컴포넌트는 이 금지 구역 규칙을 기본 제약으로 가져가야 한다.

### 4.6 복귀 루프는 후속 방향성으로 명시
- `02_시스템설계서.md`, `01_기능설계서.md`, `11_용어사전_및_가정사항.md`에 복귀 루프가 후속 설계 방향성으로 추가되었다.
- 현재 선택지는 다음 네 가지다.
  - 복귀 배너
  - 복귀 미션
  - 최근 최고 기록 회상
  - Daily Challenge 재진입 유도

개발 영향
- 이번 변경으로 즉시 구현해야 하는 것은 아니다.
- 다만 후속 확장을 고려해 관련 데이터/이벤트 구조를 닫아버리지 말아야 한다.

### 4.7 출석과 연속 방문 구분 추가
- `01_기능설계서.md` 기준으로 아래가 구분되었다.
  - 출석 보상: 당일 방문 또는 첫 플레이 기준의 단기 보상
  - 연속 방문: 업적 또는 장기 목표 축

개발 영향
- 미션/보상/업적 테이블과 로직을 설계할 때 두 개념을 같은 카운터로 처리하면 안 된다.

## 5. 테스트 문서 영향
- `07_테스트계획서.md`에 다음 테스트 항목이 추가 또는 강화되었다.
  - `TC-017A` 첫 3판 최고 기록 강조 연출
  - `TC-028A` 360x800 모바일 세로 대체 뷰포트
  - `TC-034` 분석 이벤트 5종 전체 확인
  - `TC-036` 광고 금지 구역 보호
- `08_기능별_테스트수행_MVP.md`도 위 기준에 맞게 단계별 테스트 연결이 바뀌었다.

개발 영향
- 선택자, 이벤트, 모바일 레이아웃은 이제 QA 후반이 아니라 초기 구현 산출물로 같이 준비해야 한다.

## 7. 개발팀 액션 요청
1. D1 계획에서 모바일 세로 기본 레이아웃을 완료 조건으로 반영한다.
2. 분석 이벤트 5종을 코어 루프 구현과 동시에 붙인다.
3. 결과 화면 컴포넌트에 `첫 3판 PB 강조 상태`를 반영한다.
4. 랭킹 정렬 로직과 테스트 데이터는 확정된 Marathon 동점 규칙으로 수정한다.
5. 향후 광고/프로모션 UI는 코어 루프 금지 구역 규칙을 기본 제약으로 삼는다.
6. 출석 보상과 연속 방문 업적을 동일 로직으로 합치지 않는다.

## 8. 최종 요약
- 이번 설계 변경의 핵심은 `모바일`, `분석`, `접근성`을 앞당기고, `랭킹 규칙`, `초기 성취감`, `광고 보호 규칙`을 더 엄격하게 고정한 것이다.
- 개발팀은 이번 변경을 단순 문서 정리로 보지 말고, D1~D3 구현 순서와 테스트 산출물에 직접 반영해야 한다.
