# 개발자전달 UI화면설계 요약

## 1. 목적
- 본 문서는 `23_UI화면설계서.md`를 개발자에게 전달할 때 함께 보내는 요약 인계 문서다.
- 목적은 화면 구현 기준, 우선순위, 금지 해석을 짧게 정리해 바로 작업 착수 가능하게 하는 것이다.

## 2. 전달 대상 문서
- `23_UI화면설계서.md`
- `05_화면흐름도_및_사용자시나리오.md`
- `01_기능설계서.md`

## 3. 이번 전달의 핵심 결론
- 현재 공개 웹 화면은 기술 검증용 구성과 제품 화면 구성이 혼재되어 있다.
- 앞으로 사용자 화면 구현은 `23_UI화면설계서.md`를 우선 기준으로 맞춘다.
- 즉, 개발 편의상 배치한 카드, 내부 설명, 임시 영문 문구, 디버그성 텍스트는 사용자 UI 기준에서 제거 대상이다.
- 추가로, 화면 톤 자체를 `정보 카드형 웹`에서 `흥행형 캐주얼 게임 서비스형 UI`로 올려야 한다.

## 4. 개발자가 바로 반영해야 할 핵심 변경
### 4.1 랜딩
- `바로 시작`이 첫 화면에서 가장 강한 Primary CTA여야 한다.
- Hero 장면 또는 마스코트/오브젝트 쇼케이스가 있어야 한다.
- `모드 선택`, `랭킹`은 같은 첫 화면에서 보여야 하지만 보조 위계여야 한다.
- Daily Challenge 프리뷰는 보조 카드로 내려야 한다.
- `최근 성과`는 단순 문장보다 배지, 불꽃, 갱신 마크 같은 보상 언어로 표현한다.
- 기술 검증용 카드와 개발 안내 텍스트는 제거한다.

### 4.2 게임 화면
- 모바일 세로 기준으로 `상단 상태 바 -> HUD -> 보드 중심 -> 하단 고정 조작 도크` 구조로 정렬한다.
- Hold와 Next는 보드 양옆 상단에 붙여 관계가 바로 읽히게 한다.
- HUD, 보드, 조작부가 스크롤 없이 동시에 보여야 한다.
- 점수, 콤보, 레벨은 상승 중인 감각이 들도록 숫자 스타일과 미세 모션을 가진다.
- 조작 버튼은 폼 버튼이 아니라 게임패드형 시각 언어를 가진다.
- 디버그성 상태 텍스트와 내부 데이터 노출은 제거한다.

### 4.3 결과 화면
- `재도전`이 가장 강한 CTA여야 한다.
- 결과 Hero는 성과 화면처럼 크게 보여야 한다.
- CTA 4개는 첫 화면에서 모두 보여야 한다.
- 첫 3판 개인 최고 기록 강조 상태를 수용할 수 있게 Hero 카드 상태를 분리한다.
- 공유 CTA는 자랑 욕구를 자극하는 보상 언어를 가져야 한다.
- 첫 CTA 영역에는 광고/프로모션/실험 배너를 배치하지 않는다.

### 4.4 설정/닉네임/Daily 상세
- 설정은 전용 복잡한 페이지보다 바텀시트/간결 패널 우선이다.
- 닉네임 등록은 결과 흐름을 끊지 않는 모달/바텀시트 우선이다.
- Daily 상세는 목표, 진행도, 보상, 시작 CTA가 한 화면에 읽히게 구성한다.
- Daily, 랭킹, 성과, 보상은 전용 오브젝트/배지 체계를 가진다.

### 4.5 비주얼 방향
- 독자 IP 전제의 시각 언어를 만든다.
- 전용 마스코트 또는 감정 표현 오브젝트군을 도입한다.
- 화면 대부분을 동일 톤 카드 반복으로 구성하지 않는다.
- 랜딩 Hero, 모드 카드, 결과 Hero, 랭킹 Top 3는 전면 재설계 대상으로 본다.

## 5. 우선 구현 순서
1. 랜딩 화면 위계 재정렬
2. 게임 화면 모바일 세로 레이아웃 고정
3. 결과 화면 CTA 구조와 Hero 카드 상태 정리
4. 모드 선택 화면 카드형 정보 구조 정리
5. 랭킹 화면 탭/세그먼트/내 기록 카드 정리
6. 설정, 닉네임 등록, Daily 상세를 보조 흐름으로 마감

## 6. 최소 완료 기준
- `390x844`, `360x800`에서 랜딩, 게임, 결과 화면이 스크롤 없이 핵심 요소를 보여야 한다.
- 랜딩 첫 화면에서 `바로 시작`, `모드 선택`, `랭킹`이 모두 보인다.
- 게임 화면에서 HUD, 보드, 조작부가 동시에 보인다.
- 결과 화면에서 `재도전`, `랭킹`, `공유`, `기록 저장` 4개 CTA가 첫 화면에 모두 보인다.
- 사용자 화면에 기술 검증용 카드, 내부 상태 설명, 임시 영문 문구가 남아 있지 않다.
- 화면이 `문서형 정보 앱`처럼 보이지 않고, `게임 서비스형 시각 계층`을 가진다.

## 7. 금지 해석
- 현재 임시 구현을 기준 화면으로 간주하지 않는다.
- 반응형이 된다는 이유만으로 모바일 세로 완료로 판정하지 않는다.
- 결과 화면에서 `재도전`보다 다른 CTA가 더 강하게 보여서는 안 된다.
- 플레이 중 또는 첫 결과 CTA 영역에 광고/프로모션 요소를 넣지 않는다.
- 화면 대부분을 동일한 카드 톤으로 반복하지 않는다.
- 공식 테트리스 트레이드드레스처럼 보이는 모사를 하지 않는다.

## 8. 최종 전달 문안
- UI 화면설계 기준이 새로 고정되었다. 기존 구현 화면에는 기술 검증용 카드, 임시 문구, 개발 편의 중심 배치가 섞여 있으므로, 이제부터는 `23_UI화면설계서.md`를 화면 구현의 우선 기준으로 적용해 달라. 핵심 방향은 `즉시 플레이`, `모바일 세로 우선`, `빠른 재도전`, `코어 플레이 비침범`이다.
- 특히 랜딩은 `바로 시작` CTA가 최우선이어야 하고, 게임 화면은 `보드 중심 + 상단 HUD + 하단 고정 조작 도크` 구조를 따라야 하며, 결과 화면은 `재도전`이 가장 강한 CTA가 되도록 정리해야 한다. 기술 검증용 카드, 내부 상태 설명, 임시 영문 문구는 사용자 화면에 남기지 말고, 광고/프로모션은 플레이 중 및 첫 결과 CTA 영역을 침범하면 안 된다.
