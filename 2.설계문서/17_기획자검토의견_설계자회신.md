# 기획자검토의견 설계자회신

## 1. 목적
- 본 문서는 `1.기획서/01_기획자_설계검토의견.md`에 대한 설계자의 회신서다.
- 목적은 기획자 의견별 수용 여부를 명확히 하고, 즉시 반영 범위와 후속 범위를 구분해 전달하는 것이다.

## 3. 총평 회신
- 기획자의 총평인 `조건부 적합` 판단은 타당하다.
- 설계 기준에서 핵심 기획 의도인 `게스트 즉시 플레이`, `짧은 재도전 루프`, `3개 MVP 모드`, `랭킹/개인기록`, `모바일 웹 대응`, `운영 가능성`, `IP 리스크 방어`는 이미 주요 축으로 반영되어 있다.
- 다만 기획자가 지적한 항목 중 일부는 현재 설계 문서에 방향만 있고, 제품 규칙 또는 단계 우선순위 수준으로 더 명확히 고정할 필요가 있다.

## 4. 항목별 회신
### 4.1 모바일/접근성/분석이 일정상 뒤로 밀린 문제
판단:
- `수용`

회신:
- 기획자 의견이 맞다.
- `RQ-304`, `RQ-801 ~ RQ-803`, `RQ-901 ~ RQ-903`은 모두 핵심 성공요인에 직접 연결되는 요구사항인데, 현재 `06_MVP개발계획서.md`에서는 Phase 5에 지나치게 몰려 있다.
- 이 배치는 `출시 전 품질 고도화` 관점에서는 자연스럽지만, 모바일 웹이 핵심 유입 채널이라는 기획 전제를 충분히 반영한 일정은 아니다.

반영 방침:
- 모바일 세로 기본 레이아웃 검증은 최소 Phase 1~2 완료 조건으로 당긴다.
- 접근성 최소 항목은 Phase 3 이전 범위로 앞당긴다.
- 핵심 퍼널 분석 이벤트는 코어 플레이 루프 구현과 함께 반영하는 방향으로 설계 문서를 보정한다.

### 4.2 Marathon 동점 처리 기준 문구 모호성
판단:
- `수용`

회신:
- 현재 기능설계서의 `동점 시 생존 시간 짧은 순 아님, 더 적은 총 입력 수 우선` 문구는 단일 해석이 되지 않는다.
- 기획, 개발, QA, 운영이 동일하게 판정할 수 있도록 완전한 비교 규칙이 필요하다.

반영 방침:
- Marathon 랭킹 정렬 기준은 `점수 -> 생존 시간 -> 총 입력 수 -> 달성 시각`의 완전한 순서로 다시 명시한다.
- 오름차순/내림차순도 각 항목별로 문장 안에 직접 고정한다.

### 4.3 첫 3판 개인 최고 기록 강화 연출 부족
판단:
- `수용`

회신:
- `RQ-306`은 현재 추적표에 반영돼 있으나, 기능설계서에는 `무엇을 어떻게 강화하는지`가 구체 규칙 수준으로 내려오지 않았다.
- 현재 문서는 개인 최고 기록 노출은 정의했지만, 초기 3판의 감정적 보상 설계는 아직 약하다.

반영 방침:
- 첫 3판 동안 개인 최고 기록 갱신 시 일반 결과 화면과 다른 강조 규칙을 기능설계서에 추가한다.
- 최소 범위는 다음 3가지다.
  - 강조 카피
  - 결과 카드 시각 강조
  - 재도전 CTA와의 연결 규칙

### 4.4 복귀 루프 UX 부재
판단:
- `부분 수용`

회신:
- 기획자 지적대로 현재 시스템설계서에는 `Dormant Player Segment` 배치만 있고, 사용자가 실제로 무엇을 보게 되는지는 제품 설계 수준으로 정의돼 있지 않다.
- 다만 이 항목은 현재 MVP blocker로 보지는 않는다.
- 로그인 없는 게스트 우선 구조인 현재 MVP에서는 복귀 채널을 정교하게 운영하기보다, 먼저 즉시 플레이와 데일리 재방문 루프를 닫는 편이 맞다.

반영 방침:
- 본 항목은 `후속 설계 명시 강화`로 처리한다.
- 오픈 이슈 수준을 넘어서, 후속 범위의 방향성 항목으로 `복귀 배너`, `복귀 미션`, `최근 최고 기록 회상`, `Daily Challenge 재진입 유도` 중 어떤 축으로 가져갈지 선택 가능한 설계 주제로 정리한다.

### 4.5 향후 수익화 삽입 지점 보호 규칙 부족
판단:
- `수용`

회신:
- `11_용어사전_및_가정사항.md`에는 `결제와 광고는 MVP 범위에 포함하지 않는다. 단, 배치 위치 정책은 문서상 정의한다.`고 되어 있으나, 실제 금지 구역 규칙은 아직 충분히 제품 규칙으로 고정되지 않았다.
- 기획자 지적대로 나중에 운영 단계에서 코어 루프를 훼손하지 않도록 금지 구역을 먼저 박아두는 것이 맞다.

반영 방침:
- 최소한 다음 항목은 설계 문서에 명시적으로 고정한다.
  - 플레이 도중 광고 금지
  - 첫 결과 화면의 첫 CTA 영역 침범 금지
  - 재도전 직전 강제 광고 금지

## 5. 권장 추가 보강 항목에 대한 회신
### 출석과 연속 방문의 구분
판단:
- `수용`

회신:
- 재방문 동기를 설계할 때 `출석`과 `연속 방문`은 다른 메커니즘이므로 구분하는 편이 맞다.
- 데일리 보상 구조와 업적 구조의 역할을 나눠 서술하도록 보강하겠다.

### 공유 결과 페이지의 상위 퍼센트 강조
판단:
- `부분 수용`

회신:
- 경쟁 유도 측면에서 유효한 제안이다.
- 다만 MVP 기준에서는 랭킹 순위 또는 상위 비율 중 하나만 우선 노출해도 충분하다.
- 본 항목은 공유 결과 설계 보강 시 선택 가능한 우선 표시값으로 정리하겠다.

### Daily Challenge 난이도 운영 원칙
판단:
- `수용`

회신:
- 운영자가 난이도 편차를 과도하게 만들지 않도록 최소 운영 가이드가 필요하다.
- 이 항목은 운영 문서 또는 챌린지 설계 보조 규칙으로 추가하는 것이 타당하다.

### 모바일 조작의 좌우 손잡이 옵션
판단:
- `후속 범위 수용`

회신:
- 문제 제기는 타당하지만 MVP 즉시 반영 범위로는 보지 않는다.
- `오픈 이슈` 또는 `후속 입력 확장 구조` 항목으로 남기는 것이 맞다.

## 6. 최종 판정 회신
### 판정
- `조건부 적합` 판단을 수용한다.

### 이유
- 핵심 기획 의도는 현재 설계에 대부분 반영돼 있다.
- 다만 아래 5개 항목은 설계 문서에서 더 명확히 고정해야 한다.
  - 모바일/접근성/분석의 단계 우선순위
  - Marathon 동점 처리 규칙
  - 첫 3판 성취감 강화 규칙
  - 복귀 루프 방향성
  - 광고/수익화 금지 구역

## 7. 설계자 액션 정리
1. `06_MVP개발계획서.md`에서 모바일/접근성/분석 최소 범위를 앞당겨 재배치한다.
2. `01_기능설계서.md`의 Marathon 동점 처리 규칙을 단일 해석 가능 문구로 수정한다.
3. `01_기능설계서.md`에 첫 3판 개인 최고 기록 강화 연출 규칙을 추가한다.
4. 복귀 사용자 경험은 후속 설계 항목으로 구체화한다.
5. 광고/수익화 금지 구역을 설계 문서에 명시한다.

## 8. 기획자 전달용 한 줄 회신
- 기획자 의견은 대체로 타당하며, 핵심 지적 5건 중 4건은 즉시 수용하고 1건은 후속 범위로 구조화해 반영하는 방향으로 회신한다.
