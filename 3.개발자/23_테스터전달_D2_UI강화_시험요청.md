# D2 UI 강화 제3자 시험 요청

## 1. 목적
- 본 문서는 D2 UI 범위 `FE-008 ~ FE-013`에 대한 제3자 재시험을 요청하기 위한 문서다.
- 이번 시험은 기존 UI 1차 구조 검증 범위에 더해 `FE-012` 랜딩 Hero 캐주얼 강화와 `FE-013` 모드 선택 포스터형 재디자인 반영 여부를 함께 확인한다.

## 2. 개발 측 반영 범위
- `FE-008 ~ FE-011`
  - 랜딩 CTA 위계
  - 결과 Hero/CTA 구조
  - 랭킹 기본 구조
  - 설정, 닉네임, Daily 바텀시트
- `FE-012`
  - 랜딩 Hero를 보상 스탬프, 쇼케이스 카드, 마스코트형 오브젝트 중심으로 재구성
  - `start-button`, `mode-button`, `ranking-button` first-fold 유지
  - Daily/개인 성과 보조 카드의 시선 유도 강화
- `FE-013`
  - 모드 선택을 포스터형 비주얼로 재구성
  - 모드별 감정 카피, tempo, reward badge, 선택 가이드 추가
  - `mode-card-*`, `mode-start-button-*` 선택자 유지

## 3. 개발 측 사전 검증 결과
- 워크스페이스 루트 기준 아래 명령 Pass
  - `pnpm test`
  - `pnpm build`
  - `pnpm test:e2e`

## 4. 시험 요청 범위
- 랜딩
  - `start-button`, `mode-button`, `ranking-button`이 모바일 세로 first-fold 내 모두 보이는지 확인
  - `start-button`이 가장 강한 CTA인지 확인
  - 랜딩 Hero가 정보 카드형이 아니라 보상/성과 중심 구조로 보이는지 확인
  - 임시 영문 문구나 개발용 상태 문구가 사용자 첫 화면에 남지 않았는지 확인
- 모드 선택
  - `mode-button`으로 진입 가능 여부
  - 3개 모드 카드가 포스터형 비주얼, 감정 카피, reward badge를 가지는지 확인
  - 선택 후 `이 모드 시작` 버튼으로 해당 모드 시작 가능 여부
- 랭킹
  - `ranking-button`으로 랭킹 화면 진입 가능 여부
  - 모드 탭, 기간 세그먼트, 내 기록 카드, 리스트 기본 구조 노출 확인
- 결과 화면
  - `retry-button`이 가장 강한 CTA인지 확인
  - CTA 4개가 모바일 세로 first-fold에서 모두 보이는지 확인
  - Hero 주요 지표와 상태 배지가 가려지지 않는지 확인
- 바텀시트
  - 설정, 닉네임 등록, Daily 상세가 바텀시트로 열리는지 확인
  - 닫기 후 기존 흐름이 유지되는지 확인

## 5. 권장 수행 절차
1. 워크스페이스 루트에서 `pnpm test` 실행
2. 1번 성공 후 `pnpm build` 실행
3. 2번 성공 후 `pnpm test:e2e` 실행
4. 브라우저에서 공개 웹을 열고 모바일 세로 `390x844`, `360x800` 기준으로 수동 확인 수행
5. 결과를 신규 테스트결과서에 기록

## 7. 판정 기준
- `pnpm test`, `pnpm build`, `pnpm test:e2e`가 모두 종료 코드 `0`이면 Pass
- 랜딩과 모드 선택이 설계 의도대로 캐주얼 게임 서비스 톤과 시선 유도 구조를 가지면 Pass
- 결과 화면 CTA 4개와 바텀시트 흐름이 모바일 세로 기준으로 문제없이 노출되면 Pass
- 위 항목이 모두 Pass면 `FE-008 ~ FE-013` 범위는 제3자 시험 Pass로 판정 가능

## 8. 참고 사항
- `FE-014`, `FE-015`는 이번 시험 범위에 포함하지 않는다.
- `SC-04 일시정지 오버레이`, 첫 3판 PB 강조 상태, 실제 기록 저장 연동은 이번 시험 범위에 포함하지 않는다.
- 이번 결과가 정리되기 전까지 다음 구현은 `FE-014` 착수 보류 상태다.
