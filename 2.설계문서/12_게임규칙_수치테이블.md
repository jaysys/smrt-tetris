# 게임규칙 수치테이블

## 1. 목적
게임 엔진 구현, 서버 검증, 단위 테스트, QA 재현에 사용되는 정량 규칙을 고정한다.

## 1.1 외부 참조 기준
- 게임 화면 안에서 보이는 블록 생성, 이동, 회전, Hold, Ghost 규칙의 외부 참조 기준은 `19_게임룰_외부참조기준.md`를 따른다.
- 본 문서는 그 외부 참조 기준을 실제 구현용 수치와 표로 고정한 단일 기준 문서다.

## 1.2 외부 참조 기반과 프로젝트 정책의 구분
- 아래 항목은 외부 참조 기반으로 고정한다.
  - 7-bag 블록 생성
  - SRS 회전 상태
  - SRS 벽차기
  - Hold 초기 상태와 턴당 1회 제한
  - Ghost Piece 기본 제공
- 아래 항목은 프로젝트 정책으로 유지한다.
  - DAS/ARR
  - 락 딜레이
  - 점수 수치
  - 레벨별 중력 간격
  - HUD 갱신 연출

## 2. 보드 및 스폰 기준
실제 구현의 단일 스폰 기준은 `2.2 블록별 초기 좌표와 회전 상태` 표다.
본 절은 보드 크기와 스폰 실패 판정의 상위 요약만 제공한다.

| 항목 | 값 |
|---|---|
| 가시 보드 크기 | 10 x 20 |
| 상단 스폰 버퍼 | 2행 |
| 스폰 실패 판정 | 신규 블록의 초기 점유 셀 중 1개라도 기존 블록과 충돌 시 실패 |

## 2.1 블록 정의 기준
- 좌표 기준점은 보드 좌상단을 `(0,0)`으로 둔다.
- 블록 매트릭스는 4x4 로컬 좌표 기준으로 정의한다.
- 초기 회전 상태는 `0`, 시계 방향 회전 상태는 `R`, 180도는 `2`, 반시계 방향은 `L`이다.
- 스폰 기준 보드 좌표는 로컬 좌표의 좌상단 오프셋을 더해 계산한다.
- 스폰 방향은 Guideline/SRS 기준을 따른다.
  - 모든 블록은 수평 스폰 상태를 기준으로 시작한다.
  - `J`, `L`, `T`는 flat-side first 기준이다.
  - `I`, `O`는 중앙 기준으로 스폰하고, `J/L/S/T/Z`는 좌측 반칸 보정이 없는 정수 좌표 기준으로 좌측 정렬 스폰한다.

## 2.2 블록별 초기 좌표와 회전 상태
| 블록 | 초기 상태 | 스폰 오프셋 `(x,y)` |
|---|---|---|
| I | `0` | `(3,-1)` |
| O | `0` | `(4,-1)` |
| T | `0` | `(3,-1)` |
| S | `0` | `(3,-1)` |
| Z | `0` | `(3,-1)` |
| J | `0` | `(3,-1)` |
| L | `0` | `(3,-1)` |

## 2.3 블록별 셀 좌표 집합
좌표는 각 회전 상태의 4x4 로컬 매트릭스 내 점유 셀이다.

### I 블록
- `0`: `(0,1) (1,1) (2,1) (3,1)`
- `R`: `(2,0) (2,1) (2,2) (2,3)`
- `2`: `(0,2) (1,2) (2,2) (3,2)`
- `L`: `(1,0) (1,1) (1,2) (1,3)`

### O 블록
- `0/R/2/L`: `(1,0) (2,0) (1,1) (2,1)`

### T 블록
- `0`: `(1,0) (0,1) (1,1) (2,1)`
- `R`: `(1,0) (1,1) (2,1) (1,2)`
- `2`: `(0,1) (1,1) (2,1) (1,2)`
- `L`: `(1,0) (0,1) (1,1) (1,2)`

### S 블록
- `0`: `(1,0) (2,0) (0,1) (1,1)`
- `R`: `(1,0) (1,1) (2,1) (2,2)`
- `2`: `(1,1) (2,1) (0,2) (1,2)`
- `L`: `(0,0) (0,1) (1,1) (1,2)`

### Z 블록
- `0`: `(0,0) (1,0) (1,1) (2,1)`
- `R`: `(2,0) (1,1) (2,1) (1,2)`
- `2`: `(0,1) (1,1) (1,2) (2,2)`
- `L`: `(1,0) (0,1) (1,1) (0,2)`

### J 블록
- `0`: `(0,0) (0,1) (1,1) (2,1)`
- `R`: `(1,0) (2,0) (1,1) (1,2)`
- `2`: `(0,1) (1,1) (2,1) (2,2)`
- `L`: `(1,0) (1,1) (0,2) (1,2)`

### L 블록
- `0`: `(2,0) (0,1) (1,1) (2,1)`
- `R`: `(1,0) (1,1) (1,2) (2,2)`
- `2`: `(0,1) (1,1) (2,1) (0,2)`
- `L`: `(0,0) (1,0) (1,1) (1,2)`

## 6. 벽차기 오프셋 테이블
다음 테이블은 SRS 기준 회전 테스트 순서다. 각 후보를 순서대로 검사해 최초 성공 위치를 채택한다.

주의
- 아래 벽차기 표의 좌표는 SRS 문헌 관례에 따라 `+x=오른쪽`, `+y=위쪽`을 사용한다.
- 실제 구현에서 보드 좌표가 `y 아래 방향 증가` 체계라면, 벽차기 적용 시 `y` 부호를 반전해 사용해야 한다.

### 6.3 `O` 블록용
- `O` 블록은 별도 벽차기 없이 `(0,0)` 고정으로 처리한다.

### 6.4 회전 실패 규칙
- 기본 회전 후 Test 1~5가 모두 실패하면 회전은 실패한다.
- 회전 실패 시 블록 위치와 회전 상태는 모두 유지된다.

## 6.6 스폰 실패 판정 예시
- 예시 상황: 보드 상단 2행과 3행의 `(4,0)`, `(5,0)`, `(4,1)`, `(5,1)` 셀이 이미 점유된 상태에서 O 블록을 스폰하려는 경우
- 결과: O 블록 초기 좌표와 즉시 충돌하므로 스폰 실패, 게임 오버 처리

## 8. 판정 순서
1. 하드 드롭 또는 자연 낙하 후 블록 착지
2. 락 딜레이 종료 또는 즉시 고정
3. 라인 완성 수 계산
4. Perfect Clear 여부 판정
5. Back-to-Back 여부 판정
6. Combo 누적 여부 판정
7. 점수 계산
8. HUD 갱신
9. 다음 블록 스폰
10. 스폰 실패 시 게임 오버

## 9. Perfect Clear / Back-to-Back / Combo 기준
- Perfect Clear: 라인 제거 후 보드의 점유 셀이 0개면 성립
- Back-to-Back: 직전 유효 제거가 `Quad` 또는 `Perfect Clear`일 때 유지
- Combo: 연속 턴에서 라인 제거가 끊기지 않으면 1씩 증가하고, 라인 제거 없는 턴에서 0으로 초기화
