# 설계자 재회신 요청

## 1. 목적
본 문서는 `04_설계자_검토회신.md` 기준으로 설계 문서 재반영 결과를 다시 검토한 뒤, 현재 시점에서 설계자에게 추가 재회신이 필요한지 여부를 정리한 최신 상태 문서다.

## 2. 재검토 결론
이전 재회신 요청의 핵심 지적 사항은 이번 반영으로 대부분 해소되었다.

확인 결과:
- 운영 API 상세 계약 보강: 반영 확인
- QA 시간 제어 방식 단일화: 반영 확인
- 게임규칙 수치테이블의 스폰 기준 단일화: 반영 확인

현재 기준으로는 설계 문서 자체에 대해 설계자에게 다시 수정 요청해야 할 미진 항목은 발견하지 못했다.

## 3. 반영 확인 내용
### 3.1 운영 API 상세 계약
`2.설계문서/03_API명세서.md`에서 다음 항목의 상세 계약 보강이 확인되었다.
- `PATCH /admin/daily-challenges/{challengeId}`
- `POST /admin/seasons`
- `POST /admin/announcements`
- `POST /admin/fraud-flags/{flagId}/actions`
- `GET /admin/config-versions`

확인된 반영 내용:
- 요청 필드 상세 표
- 응답 필드 상세 표
- Required / Nullable / 제약 조건
- 중첩 구조 또는 배열 구조
- 대표 오류 응답 예시

판단:
- 이전의 `부분완료` 상태에서 `완료`로 상향 가능하다.

### 3.2 QA 시간 제어 방식
`2.설계문서/14_QA실행가이드.md`에서 표준 시간 제어 수단이 `TEST_CLOCK_ISO_UTC` 환경변수 하나로 정리된 것을 확인했다.

확인된 반영 내용:
- 관리자 테스트 시계 설정 API는 표준 경로에서 제외
- 로컬 QA와 자동화 QA 모두 동일 기준 사용
- 확인 API, 화면, DB 지점 유지

판단:
- 이전의 `부분완료` 상태에서 `완료`로 상향 가능하다.

### 3.3 게임규칙 수치테이블
`2.설계문서/12_게임규칙_수치테이블.md`에서 스폰 기준의 상대 표현이 제거되고, `2.2 블록별 초기 좌표와 회전 상태` 표가 구현의 단일 기준임이 명시된 것을 확인했다.

확인된 반영 내용:
- `기본 스폰 X`, `기본 스폰 Y` 제거
- 상위 요약과 실제 구현 기준 구분
- 블록별 절대 스폰 오프셋 표 유지

판단:
- 이전의 문서 내 이중 기준 문제는 해소되었다.

## 4. 현재 남아 있는 보류 항목
### 4.1 운영배포체크리스트의 실제 명령 검증
이 항목은 설계 미진이 아니라 현재 작업공간 한계로 인해 보류 상태다.

현재 상태:
- `2.설계문서/10_운영배포체크리스트.md`는 `예정 명령 규약`, `예시 명령`으로 적절히 조정되어 있다.
- 그러나 현재 프로젝트 폴더에는 실제 코드 저장소와 `package.json`, `pnpm-workspace.yaml`, `turbo.json`이 없어 명령 실재 여부를 검증할 수 없다.

판단:
- 설계자에게 지금 추가 수정 요청할 사항은 아니다.
- 실제 개발 저장소 생성 후 개발자가 다시 검증해야 할 운영 정합성 항목이다.

## 5. 최종 판단
현재 설계 문서는 이번 반영으로 재검토 요청 사항을 실질적으로 해소했다.

따라서:
- 설계자에게 즉시 추가 재회신을 요구할 사항은 없음
- 설계 문서 기준으로는 개발 착수 가능
- 남은 확인 과제는 코드 저장소 생성 이후 운영 명령과 실제 스크립트 정합성 검증

## 6. 전달 권장 방식
설계자에게 전달이 필요하다면 아래처럼 전달하면 충분하다.

- 이전 재회신 요청 사항은 이번 반영으로 해소된 것으로 확인함
- 추가 설계 수정 요청은 현재 없음
- 단, 운영 명령 실재 여부는 코드 저장소 생성 후 개발 단계에서 별도 검증 예정
