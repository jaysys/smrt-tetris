## 2. 작성 원칙
- 설계 테스트 ID 체계와 동일하게 `TC-001 ~ TC-036`, `TC-017A`, `TC-028A`를 사용한다.
- 각 케이스는 화면, API, 이벤트, 데이터 중 필요한 판정을 함께 수행한다.
- 엔진 관련 케이스는 외부 참조 기준과 수치테이블을 기대결과의 우선 근거로 삼는다.
- 증빙은 스크린샷, 영상, 이벤트 로그, 응답 JSON, DB/스토리지 확인 중 최소 1개 이상 확보한다.

## 3. 공통 실행 규칙
- 기본 환경은 웹 `http://localhost:3000`, API `http://localhost:3001/v1`, 운영 `http://localhost:3000/admin`를 사용한다.
- 날짜 경계 테스트는 `TEST_CLOCK_ISO_UTC`를 사용한다.
- 결과 제출 재시도 케이스는 `Idempotency-Key`를 반드시 포함한다.
- 모바일 케이스는 `390x844`, `360x800` 캡처를 각각 확보한다.
- 이벤트 케이스는 `landing_view`, `quick_start_click`, `game_start`, `game_finish`, `retry_click` 5종을 정확한 이름으로 확인한다.

## 4. P0 상세 테스트케이스

### TC-001 랜딩 진입
- 우선순위: `P0`
- 관련 영역: 랜딩, `GET /bootstrap`
- 개발 단계: `D0`, `D1`
- 전제조건: 웹과 API가 실행 중이다.
- 수행절차:
  1. 새 브라우저 세션에서 메인 URL에 접속한다.
  2. 랜딩 첫 렌더 시간을 기록한다.
  3. `GET /v1/bootstrap` 요청과 응답을 확인한다.
- 기대결과:
  - 랜딩이 3초 이내 표시된다.
  - `바로 시작`, `모드 선택`, `랭킹` CTA가 보인다.
  - 응답에 `guestToken`, `defaultMode`, `dailyChallenge`, `settings`가 포함된다.

### TC-002 게스트 즉시 플레이

- 개발 단계: `D1`
- 전제조건: 미로그인 상태다.
- 수행절차:
  1. 랜딩에서 `바로 시작`을 누른다.
  2. 로그인/회원가입 유도 노출 여부를 확인한다.
  3. 세션 생성 요청과 응답을 확인한다.
- 기대결과:
  - 인증 없이 게임 화면으로 이동한다.
  - `POST /v1/game-sessions`가 성공한다.
  - 시작 클릭 후 30초 이내 플레이 가능 상태가 된다.

  1. 중앙에서 회전 입력을 수행한다.

### TC-008 Hold 기능
- 우선순위: `P0`
- 관련 영역: Hold, next queue
- 개발 단계: `D1`
- 전제조건: Hold 기능이 활성화되어 있다.
- 수행절차:
  1. 현재 블록에서 Hold를 1회 수행한다.

  - Hold로 다시 나온 블록은 기존 회전 상태 유지가 아니라 스폰 방향으로 재정렬된다.

- 관련 영역: Ghost
- 개발 단계: `D1`

- 수행절차:

  - Ghost는 표시 전용이며 실제 충돌 실체가 아니다.

### TC-011 점수 계산

- 개발 단계: `D2`

- 수행절차:
  1. 각 라인 제거 패턴을 수행한다.
  2. 점수와 HUD 갱신 순서를 기록한다.

- 기대결과:
  - 판정 순서는 `착지 -> 고정 -> 라인 수 계산 -> Perfect Clear -> Back-to-Back -> Combo -> 점수 계산 -> HUD 갱신 -> 다음 블록 스폰`과 일치한다.
