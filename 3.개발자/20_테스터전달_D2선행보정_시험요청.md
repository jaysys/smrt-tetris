# D2 선행 보정 제3자 시험 요청

## 1. 목적
- 본 문서는 D2 본 기능 착수 전에 선행 보정 범위 `GE-003`, `FE-004`, `FE-005`, `FE-006`에 대한 제3자 시험을 요청하기 위한 문서다.
- 이번 시험이 끝나기 전까지 D2 본 기능 `TC-009 ~ TC-011` 구현은 시작하지 않는다.

## 2. 개발 측 반영 범위
- `GE-003`
  - 전이별 SRS kick table 반영
  - Perfect Clear, Back-to-Back, Combo, 중력, 소프트 드롭 규칙 보정
- `FE-004`
  - `landing_view`, `quick_start_click`, `game_start`, `game_finish`, `retry_click` 이벤트 발생
  - 브라우저 전역 `window.__tetrisAnalyticsEvents` queue 노출
- `FE-005`
  - 모바일 세로 `360x800` 기준 랜딩, 플레이, 결과 화면 레이아웃 정리
  - HUD, 터치 조작부, 결과 화면 수치 영역 겹침 제거
- `FE-006`
  - 고대비 모드 토글 추가
  - 주요 버튼 `aria-label`, `aria-live`, 대체 조작 안내 반영

## 3. 개발 측 사전 검증 결과
- 아래 명령은 모두 워크스페이스 루트에서 직렬 실행 기준 Pass다.
  - `pnpm --filter @tetris/game-engine test`
  - `pnpm test`
  - `pnpm build`
  - `pnpm test:e2e`

## 4. 시험 요청 범위
- 엔진 규칙 회귀 확인
  - 회전/벽차기 시 비정상 충돌이나 비합리적 위치 점프가 없는지 확인
  - 연속 라인 클리어 시 Combo/Back-to-Back 점수 상태가 반영되는지 확인
- 분석 이벤트 확인
  - `바로 시작`, 게임 종료, `다시 하기` 수행 후 브라우저 콘솔에서 `window.__tetrisAnalyticsEvents` 확인
  - 이벤트 이름이 아래 순서로 누적되는지 확인
    - `landing_view`
    - `quick_start_click`
    - `game_start`
    - `game_finish`
    - `retry_click`
- 모바일 세로 `360x800` 확인
  - 랜딩 화면 CTA, 플레이 HUD, 터치 조작부, 결과 화면 CTA가 겹치지 않는지 확인
  - `touch-controls`, `hold-slot`, `next-queue`, `score-value`, `result-ranking-button`이 모두 관찰 가능한지 확인
- 접근성 옵션 확인
  - `contrast-toggle` 또는 `playing-contrast-toggle`로 고대비 모드 전환 가능한지 확인
  - 주요 버튼이 키보드 포커스와 스크린리더용 이름을 가지는지 확인
  - `accessibility-guide`, `touch-help`, `rank-status`가 화면에 노출되는지 확인

## 5. 권장 수행 절차
1. 워크스페이스 루트에서 `pnpm test` 실행
2. 1번 성공 후 `pnpm build` 실행
3. 2번 성공 후 `pnpm test:e2e` 실행
4. 브라우저에서 공개 웹을 열고 모바일 세로 `360x800` 기준으로 수동 확인 수행
5. 결과를 신규 테스트결과서에 기록

## 7. 판정 기준
- 자동화 명령 `pnpm test`, `pnpm build`, `pnpm test:e2e`가 모두 종료 코드 `0`이면 Pass
- 모바일 세로 `360x800`에서 주요 UI가 겹치지 않고 조작 가능하면 Pass
- 고대비 모드 전환과 주요 접근성 속성 노출이 확인되면 Pass
- 위 항목이 모두 Pass면 `GE-003 ~ FE-006` 제3자 시험은 Pass로 판정 가능

## 8. 참고 사항
- 공식 게이트는 직렬 실행을 기준으로 한다.
- 분석 이벤트 확인은 브라우저 개발자도구 콘솔에서 `window.__tetrisAnalyticsEvents`를 조회하면 된다.
- 본 시험이 끝나기 전까지 D2 본 기능 개발은 착수하지 않는다.
