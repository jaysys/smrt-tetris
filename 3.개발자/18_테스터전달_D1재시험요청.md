# D1 재시험 요청

## 1. 목적
- 본 문서는 `4.테스터/07_D1_테스트결과서.md` 기준 D1 직접 기능 검증 Pass 이후 남아 있던 공식 게이트 실패 건에 대한 수정 반영 사실과 재시험 범위를 전달하기 위한 문서다.

## 2. 개발 측 재확인 결과
- 직접 기능 검증 `TC-002 ~ TC-008` Pass 판단은 그대로 유효하다.
- 개발 측은 공개 웹 프로덕션 빌드 안정성 문제를 보완했다.
- 수정 반영:
  - `apps/web/src/pages/500.tsx` 추가
- 최신 개발 측 재검증 결과:
  - `pnpm test` Pass
  - `pnpm build` Pass
  - `pnpm test:e2e` Pass

## 3. 재시험 요청 범위
- 직접 기능 검증은 재수행 대상이 아니다.
- 아래 2건만 재시험해 결과서를 갱신해 달라.
  - `pnpm build`
  - `pnpm test:e2e`

## 4. 실행 순서
1. 워크스페이스 루트에서 `pnpm build` 실행
2. 1번 성공 후 워크스페이스 루트에서 `pnpm test:e2e` 실행
3. 결과를 `4.테스터/07_D1_테스트결과서.md`에 반영

## 5. 판정 기준
- `pnpm build`가 종료 코드 `0`으로 완료되면 Pass
- `pnpm test:e2e`가 종료 코드 `0`이고 3개 시나리오가 모두 성공하면 Pass
- 위 2건이 모두 Pass면 D1 종합 판정은 `Pass`로 갱신 가능

## 6. 참고 사항
- 공식 게이트는 직렬 실행을 기준으로 한다.
- `pnpm build`와 `pnpm test:e2e`는 둘 다 `apps/web/.next` 산출물을 사용하므로 동시에 실행하지 않는다.
