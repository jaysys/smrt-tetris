# 개발자 전달 게임엔진 구현지시서

## 1. 목적
- 본 문서는 개발자가 테스트 게임의 코어 엔진을 구현할 때 따라야 할 단일 구현 지시서다.
- 특히 게임 화면 안에서 보이는 블록의 생성, 이동, 회전, 벽차기, Hold, Ghost, 줄 제거 규칙을 외부 참조 기준에 맞춰 정확히 구현하도록 하는 데 목적이 있다.

## 2. 구현 기준 문서
- 외부 참조 기준: `19_게임룰_외부참조기준.md`
- 기능 요구 기준: `01_기능설계서.md`
- 정량 구현 기준: `12_게임규칙_수치테이블.md`
- 시스템 검증 기준: `02_시스템설계서.md`

개발자는 위 4개 문서를 함께 보되, 실제 엔진 구현의 최종 수치 기준은 반드시 `12_게임규칙_수치테이블.md`를 따른다.

## 3. 이번 구현에서 반드시 고정할 핵심 룰
### 3.1 블록 생성
- 블록 공급은 `7-bag(Random Generator)`를 사용한다.
- 7종 블록을 1세트로 셔플한 뒤 순차 소비한다.
- 다음 블록은 5개까지 표시한다.

### 3.2 보드와 스폰
- 보드는 가시 영역 `10 x 20`을 사용한다.
- 상단 스폰 버퍼 `2행`을 둔다.
- 블록 초기 상태는 모두 `0` 상태에서 시작한다.
- 스폰 방향과 초기 좌표는 `12_게임규칙_수치테이블.md`의 표를 그대로 따른다.
- 신규 블록의 초기 점유 셀이 기존 블록과 충돌하면 즉시 스폰 실패, 게임 오버 처리한다.

### 3.3 이동과 낙하
- 좌우 이동은 충돌이 없을 때만 허용한다.
- 자연 낙하는 레벨별 중력 간격을 따른다.
- 소프트 드롭은 기본 중력의 15배 가속으로 처리한다.
- 하드 드롭은 즉시 착지 및 즉시 점수 반영으로 처리한다.

### 3.4 회전과 벽차기
- 회전 규칙은 `SRS(Super Rotation System)`를 사용한다.
- `J/L/S/T/Z`는 공통 벽차기 표를 사용한다.
- `I`는 별도 벽차기 표를 사용한다.
- `O`는 별도 벽차기 없이 `(0,0)` 기준으로 처리한다.
- 회전 테스트는 표의 순서대로 검사하고, 최초 성공 위치를 채택한다.
- 5개 테스트가 모두 실패하면 회전은 실패이며, 위치와 회전 상태를 유지한다.

### 3.5 Hold
- Hold는 기본 제공한다.
- 현재 블록이 잠기기 전까지 1턴당 1회만 허용한다.
- Hold 박스 초기 상태는 비어 있다.
- Hold로 다시 꺼낸 블록은 현재 방향을 유지하지 말고 `스폰 방향`으로 재정렬한다.

### 3.6 Ghost
- Ghost Piece는 기본값 `ON`이다.
- 현재 블록이 즉시 하드 드롭 되었을 때의 착지 위치를 표시한다.
- Ghost는 시각 보조 요소이며, 충돌 판정의 기준 블록은 실제 현재 블록이다.

### 3.7 줄 제거와 판정 순서
- 줄 완성은 블록 고정 직후 같은 판정 사이클에서 계산한다.
- 판정 순서는 반드시 아래를 따른다.
1. 착지
2. 고정
3. 라인 수 계산
4. Perfect Clear 판정
5. Back-to-Back 판정
6. Combo 판정
7. 점수 계산
8. HUD 갱신
9. 다음 블록 스폰
10. 스폰 실패 시 게임 오버

## 4. 구현 시 금지 해석
- 임의 랜덤 생성으로 블록을 공급하면 안 된다. 반드시 `7-bag`여야 한다.
- 단순 회전만 구현하고 벽차기 표를 생략하면 안 된다.
- Hold에서 이전 방향을 유지한 채 다시 꺼내면 안 된다. 반드시 스폰 방향 재정렬이다.
- `O` 블록에 일반 블록용 벽차기 표를 적용하면 안 된다.
- Ghost를 실제 충돌 오브젝트처럼 취급하면 안 된다.
- 화면에 보이는 동작 규칙을 개발 편의상 임의 단순화하면 안 된다.

## 5. 권장 구현 순서
1. 보드 좌표계와 충돌 판정 고정
2. 블록 정의와 스폰 좌표 구현
3. 좌우 이동, 자연 낙하, 하드 드롭 구현
4. SRS 회전 상태 전이 구현
5. 벽차기 표 적용 구현
6. 7-bag 공급기 구현
7. Hold, Ghost 구현
8. 줄 제거와 점수 판정 구현
9. HUD 반영과 결과 판정 연결

## 6. 최소 단위 테스트 요구
- 7-bag 한 세트 안에서 7종 블록이 중복 없이 1회씩만 나오는지 검증
- 각 블록의 초기 스폰 좌표가 표와 일치하는지 검증
- `J/L/S/T/Z`와 `I`의 벽차기 표가 전이별로 다르게 적용되는지 검증
- 회전 실패 시 위치와 상태가 유지되는지 검증
- Hold 최초 사용, Hold 재호출, 턴당 1회 제한이 맞는지 검증
- Ghost 위치가 실제 하드 드롭 착지 위치와 일치하는지 검증
- 1줄, 2줄, 3줄, 4줄 제거와 Perfect Clear 점수가 맞는지 검증
- 스폰 충돌 시 즉시 게임 오버가 되는지 검증

## 7. 개발자 전달 결론
- 테스트 게임의 화면 내 블록 동작은 이제 외부 참조 기준이 명시된 상태다.
- 개발자는 `Guideline 계열 동작`을 기준으로 구현해야 한다.
- 구현 중 임의 해석이 필요하면 새 규칙을 만들지 말고 먼저 설계 검토를 요청해야 한다.
