# 기능설계서

## 1. 문서 개요
- 문서명: 테트리스형 웹서비스 기능설계서
- 기준 문서: `1.기획서/테트리스_웹서비스_요구사항정의서.md`
- 설계 기준일: 2026-03-11
- 설계 목적: 개발자가 기능 단위 구현을 바로 시작할 수 있도록 사용자 흐름, 기능 규칙, 예외, 상태 전이를 상세 정의한다.

## 2. 설계 전제
- 본 설계는 `정식 테트리스 라이선스 미확보 상태`를 기본 전제로 한다.
- 외부 노출 명칭은 `낙하형 블록 퍼즐`, `블록`, `라인 제거`, `마라톤`, `스프린트`, `데일리 챌린지` 같은 일반 표현을 사용한다.
- 브랜드명, 로고, 블록 모양 명칭은 법무 검토 후 확정한다.
- 게임 화면 안에서 보이는 블록의 생성, 이동, 회전, Hold, Ghost 규칙의 외부 참조 기준은 `19_게임룰_외부참조기준.md`를 따른다.
- MVP 범위는 `게스트 즉시 플레이`, `Marathon`, `Sprint`, `Daily Challenge`, `랭킹`, `개인 최고 기록`, `기본 업적/미션`, `운영 설정`까지로 한다.
- 로그인은 필수가 아니며, 게스트 플레이가 기본 진입 경로다.

## 2.1 시간 기준
- DB 저장 시간은 모두 UTC 기준 `timestamptz`로 저장한다.
- 사용자 UI 표시 시간은 기본적으로 KST(UTC+09:00) 기준으로 변환 표시한다.
- Daily Challenge 갱신 시각은 매일 KST 00:00:00이다.
- 일간 랭킹 `daily`의 집계 기간은 `KST 00:00:00 ~ 23:59:59.999`다.
- 주간 랭킹 `weekly`의 집계 기간은 `매주 월요일 KST 00:00:00 ~ 다음 월요일 직전`이다.

## 5. 전체 사용자 흐름
1. 사용자가 랜딩 페이지에 진입한다.
2. `바로 시작`, `모드 선택`, `랭킹` 중 하나를 선택한다.
3. `바로 시작` 선택 시 기본 모드인 Marathon으로 즉시 게임 화면에 진입한다.
4. 첫 플레이 시 튜토리얼 힌트 3단계를 오버레이로 노출한다. 사용자는 즉시 스킵할 수 있다.
5. 게임 종료 후 결과 화면에서 `재도전`, `랭킹 보기`, `기록 저장`, `공유`, `보상 확인`을 수행한다.
6. 닉네임 미등록 사용자는 최초 기록 등록 시 닉네임 생성 모달을 본다.
7. Daily Challenge와 업적, 출석, 최고 기록 갱신이 재방문 루프를 만든다.

### 6.3 기록 검증 상태
`SUBMITTED -> VALIDATING -> ACCEPTED | REJECTED | FLAGGED`

## 7. 상세 기능 설계
## 7.1 F-01 첫 진입 및 빠른 시작
### 목적
첫 방문자가 로그인 없이 1클릭으로 실제 플레이를 시작하도록 한다.

### 진입 화면 구성
- 상단: 서비스 로고, 공지 배너, 설정 아이콘
- 메인 CTA: `바로 시작`
- 보조 CTA: `모드 선택`, `랭킹`
- 하단: 최근 최고 기록, Daily Challenge 요약, 공지 링크

### 처리 규칙
- 기본 포커스는 `바로 시작` 버튼에 둔다.
- 게스트 식별자는 첫 방문 시 브라우저 로컬 스토리지와 서버 게스트 토큰에 동시에 저장한다.
- 첫 플레이 전 회원가입은 노출하지 않는다.
- 튜토리얼은 3단계 이하로 제한한다.
- 첫 진입 시 마지막 플레이 모드 정보가 있으면 `최근 플레이 계속` 보조 버튼을 노출할 수 있다.

### 예외 처리
- 부트스트랩 API 실패 시 오프라인 불가 안내와 재시도 버튼을 노출한다.
- 설정 데이터 일부가 없어도 기본값으로 화면을 열어야 한다.

### 완료 조건
- 최초 방문 후 30초 이내 첫 게임 시작이 가능하다.
- 회원가입 없이 첫 게임 진입이 가능하다.

## 7.2 F-02 코어 게임플레이
### 목적
이동, 회전, 드롭, 라인 제거, 점수 반영이 예측 가능하고 공정하게 동작해야 한다.

### 입력 체계
| 동작 | 키보드 | 터치 UI | 비고 |
|---|---|---|---|
| 좌측 이동 | Left / A | 좌측 버튼 | 반복 입력 허용 |
| 우측 이동 | Right / D | 우측 버튼 | 반복 입력 허용 |
| 회전 | Up / X | 회전 버튼 | 시계 방향 기본 |
| 반시계 회전 | Z | 보조 회전 버튼 | 선택 기능 |
| 소프트 드롭 | Down / S | 아래 버튼 길게 | 가속 낙하 |
| 하드 드롭 | Space | 하드 드롭 버튼 | 즉시 고정 |
| Hold | C / Shift | Hold 버튼 | 턴당 1회 |
| 일시정지 | Esc / P | 일시정지 버튼 | 브라우저 포커스 아웃과 연동 |

### 게임 규칙
- 보드는 10 x 20 가시 영역과 상단 스폰 버퍼를 가진다.
- 블록 공급은 7-bag 랜덤 규칙을 사용한다.
- 회전 판정은 SRS(Super Rotation System) 기준 벽차기 규칙 테이블을 사용한다.
- `J/L/S/T/Z`는 공통 벽차기 표를 사용하고, `I`는 별도 벽차기 표를 사용하며, `O`는 별도 벽차기 없이 처리한다.
- 줄이 완성되면 같은 프레임에서 제거를 판정한다.
- 상단 제한선 위에 새 블록이 정상 배치되지 못하면 게임 오버다.
- Hold 기능은 `블록이 잠긴 뒤 다음 턴부터 재사용 가능` 정책으로 한다.
- Hold로 다시 호출된 블록은 스폰 방향 기준으로 재정렬한다.
- Ghost Piece는 기본값 `ON`으로 제공한다.
- 초보자 보조 기능은 `Ghost Piece`, `입력 가이드`, `실수 방지 진동` 세 가지로 구성한다.
- 외부 참조 출처와 채택 원칙은 `19_게임룰_외부참조기준.md`, 실제 구현 수치는 `12_게임규칙_수치테이블.md`를 단일 기준으로 사용한다.

### 점수 규칙
| 이벤트 | 점수 |
|---|---|
| Single | 100 x 현재 레벨 |
| Double | 300 x 현재 레벨 |
| Triple | 500 x 현재 레벨 |
| Quad | 800 x 현재 레벨 |
| 소프트 드롭 | 칸당 1점 |
| 하드 드롭 | 칸당 2점 |
| 콤보 | 50 x 콤보 수 x 현재 레벨 |
| Back-to-Back | 직전 난도 높은 제거 연속 시 기본 점수 1.5배 |
| Perfect Clear | 별도 보너스 2000 x 현재 레벨 |

### 난이도 상승 규칙
- Marathon은 10라인마다 레벨이 1 증가한다.
- 레벨 상승 시 자연 낙하 속도를 단계형으로 증가시킨다.
- Sprint는 속도 증가 대신 `40라인 제거 시간`만 경쟁한다.
- Daily Challenge는 운영 설정으로 목표 조건을 바꾼다.

### HUD 구성
- 좌측 상단: 현재 점수
- 좌측 중단: 제거 라인 수, 레벨
- 우측 상단: 다음 블록 5개 미리보기
- 우측 중단: Hold 블록
- 하단: 콤보, 백투백, 개인 최고 기록 대비 상태

### 브라우저 이벤트 정책
- 포커스 아웃 시 기본 정책은 자동 일시정지다.
- 새로고침 시 진행 중 점수는 폐기하고 세션 중단으로 처리한다.
- 모바일 전화 수신 또는 앱 전환 후 30초 이내 복귀 시 일시정지 해제 가능하다.

### 완료 조건
- 모든 필수 입력이 1프레임 기준 정상 반영된다.
- 라인 제거와 점수 증가가 즉시 HUD에 표시된다.
- Hold, Ghost Piece, 다음 블록 미리보기가 정상 동작한다.

## 7.3 F-03 모드 운영
### Marathon
- 목적: 최대한 오래 생존하며 고득점을 달성한다.
- 종료 조건: 게임 오버
- 랭킹 기준:
  - 1순위: 점수 내림차순
  - 2순위: 총 생존 시간 오름차순
  - 3순위: 총 입력 수 오름차순
  - 4순위: 먼저 달성한 기록 우선

### Sprint
- 목적: 40라인을 가장 빨리 제거한다.
- 종료 조건: 40라인 제거 완료
- 랭킹 기준: 기록 시간 오름차순, 동점 시 더 적은 총 입력 수 우선

### Daily Challenge
- 목적: 운영자가 지정한 일일 미션을 달성한다.
- 종료 조건: 미션 성공 또는 실패 조건 도달
- 예시 미션:
  - 3분 안에 20라인 제거
  - Hold 없이 10라인 제거
  - 특정 점수 이상 달성
- 재도전: 하루 제한 없음, 보상 수령은 1회

### 공통 규칙
- 모든 모드는 결과 화면 구조를 공통으로 사용한다.
- 모드별 랭킹 테이블은 분리한다.
- Daily Challenge는 UTC+09:00 기준 매일 00:00에 갱신한다.

### 완료 조건
- MVP 모드 3종이 각각 다른 종료 조건과 랭킹 기준을 가진다.
- Daily Challenge가 매일 자동 갱신된다.

## 7.4 F-04 결과 화면 및 재도전
### 결과 화면 필수 요소
- 최종 점수 또는 기록 시간
- 제거 라인 수, 레벨, 최대 콤보
- 개인 최고 기록 갱신 여부
- 랭킹 진입 여부
- 데일리 보상/업적 진행도
- CTA: `재도전`, `랭킹`, `공유`, `기록 저장`

### 처리 규칙
- 결과 화면 첫 CTA는 항상 `재도전`이다.
- 기록 검증 중이어도 결과 화면은 즉시 표시한다.
- 기록 검증 완료 후 랭킹 변동은 비동기 갱신 배지로 보여준다.
- 결과 화면은 게임 종료 후 `1초 이내` 표시되어야 한다.
- `재도전` 클릭 후 다음 게임 세션 시작까지 허용 시간은 `2초 이내`다.
- 첫 화면에서 CTA 4개가 스크롤 없이 보여야 한다.
- 첫 3판 이내 사용자가 개인 최고 기록을 갱신하면 일반 결과와 구분되는 강조 연출을 노출한다.
- 첫 3판 개인 최고 기록 강조 연출은 아래 3요소를 동시에 만족해야 한다.
  - 축하 카피 또는 배지
  - 결과 카드 시각 강조
  - `재도전` CTA와 인접한 위치에서 재도전 유도

### 완료 조건
- 게임 종료 후 1초 이내 결과 화면이 표시된다.
- 사용자는 결과 화면에서 추가 로딩 없이 재도전할 수 있다.
- `재도전`, `랭킹`, `공유`, `기록 저장` CTA 4개가 최초 화면에서 모두 보여야 한다.

## 7.5 F-05 개인 기록 및 닉네임 등록
### 닉네임 정책
- 2자 이상 12자 이하
- 한글, 영문, 숫자, 밑줄 허용
- 금칙어 필터 적용
- 중복 허용, 내부적으로는 `profile_id`로 식별

### 기록 정책
- 게스트도 닉네임 등록 후 공식 랭킹 반영 가능
- 닉네임 미등록 상태에서는 임시 기록으로 저장하고 결과 화면에서 등록 유도
- 개인 최고 기록은 모드별로 별도 관리

### 완료 조건
- 부적절한 닉네임은 등록되지 않는다.
- 닉네임 등록 후 해당 세션 기록이 랭킹에 연결된다.

## 7.6 F-06 랭킹
### 랭킹 범위
- 일간
- 주간
- 전체

### 표시 항목
- 순위
- 닉네임
- 점수 또는 시간
- 달성 일시
- 모드
- 본인 여부 강조

### 처리 규칙
- Marathon과 Daily Challenge는 점수 기준 내림차순
- Sprint는 시간 기준 오름차순
- 의심 기록은 기본 랭킹에서 제외하고 검증 큐로 이동
- 본인 최고 기록과 현재 랭킹 간 차이를 표시한다

### 완료 조건
- 모드별, 기간별 랭킹을 분리 조회할 수 있다.
- 비정상 기록은 일반 랭킹에 노출되지 않는다.

## 7.7 F-07 데일리 챌린지 및 미션
### 구조
- Daily Challenge 1개
- 반복 미션 3개
  - 오늘 3판 플레이
  - 최고 기록 1회 갱신
  - 랭킹 페이지 1회 조회
- 출석 보상은 `당일 방문 또는 첫 플레이` 기준의 단기 보상으로 정의한다.
- 연속 방문은 업적 또는 장기 목표 축에서 별도로 관리한다.

### 보상 정책
- Daily Challenge 완료 시 고정 보상 지급
- 반복 미션은 목표 달성 즉시 수령 가능
- 보상은 스킨 포인트, 배지 포인트, 업적 진척도로 환산
- Daily Challenge 난이도는 아래 운영 원칙을 따른다.
  - 연속 3일 이상 극단적으로 쉬운 챌린지 또는 극단적으로 어려운 챌린지를 반복하지 않는다.
  - 연속 2일 이상 동일 제약형 목표를 반복하지 않는다.
  - 주 1회 이상은 캐주얼 사용자도 달성 가능한 난이도를 포함한다.

### 완료 조건
- 데일리 챌린지와 미션 진행도가 플레이 직후 반영된다.
- 중복 보상 수령이 불가능하다.

## 7.8 F-08 업적 및 장기 목표
### MVP 업적
- 첫 플레이 완료
- Marathon 10000점 달성
- Sprint 첫 완주
- Daily Challenge 3회 완료
- 연속 3일 방문

### 처리 규칙
- 업적은 서버 기준으로 판정한다.
- 업적 달성 시 결과 화면과 프로필 요약에서 동시에 노출한다.

### 완료 조건
- 업적 달성 후 중복 달성으로 같은 보상이 반복 지급되지 않는다.

## 7.9 F-09 설정 및 접근성
### 설정 항목
- 사운드 ON/OFF
- 진동 ON/OFF
- 시각 효과 강도
- Ghost Piece ON/OFF
- 고대비 모드
- 블록 테마 선택
- 입력 가이드 표시

### 접근성 규칙
- 색상만으로 상태를 구분하지 않는다.
- HUD 최소 대비 4.5:1 이상을 목표로 한다.
- 텍스트 최소 크기는 모바일 기준 14px 상당으로 유지한다.

### 완료 조건
- 설정 변경이 즉시 반영되고 다음 세션에도 유지된다.
- 효과 축소 설정 시 불필요한 화면 흔들림이 제거된다.
- 모바일 390 x 844, 360 x 800 뷰포트에서 HUD와 조작부 겹침이 없어야 한다.

## 7.10 F-10 공유
### 공유 대상
- 결과 링크
- 결과 요약 이미지

### 표시 정보
- 모드
- 점수 또는 기록 시간
- 개인 최고 기록 여부
- 랭킹 순위 또는 상위 비율

### 처리 규칙
- 공유 링크는 만료 없는 읽기 전용 링크를 기본으로 한다.
- 공유 이미지에는 내부 식별자를 직접 노출하지 않는다.
- 공유 결과 페이지는 `랭킹 순위` 또는 `상위 비율` 중 하나를 주요 비교 정보로 우선 노출한다.

### 완료 조건
- 결과 화면에서 2탭 이내 공유가 가능하다.
- 공유 링크 접속 시 결과 요약 페이지가 정상 열린다.

## 7.11 F-11 운영 백오피스
### 필수 화면
- Daily Challenge 설정
- 시즌/랭킹 기간 설정
- 보상 수치 설정
- 공지사항 관리
- 제재 대상 조회

### 처리 규칙

- 반영 시작 시각을 예약할 수 있다.
- 공지사항은 즉시 게시와 예약 게시를 지원한다.

### 완료 조건
- 운영자가 코드 배포 없이 Daily Challenge를 교체할 수 있다.
- 운영자가 비정상 기록을 차단 상태로 전환할 수 있다.

## 7.12 F-12 부정행위 및 운영 보호
### 탐지 대상
- 비현실적 입력 빈도
- 비정상 기록 시간
- 클라이언트 조작 흔적
- 반복 욕설 닉네임 변경

### 대응 정책
- 1차: 자동 플래그
- 2차: 랭킹 제외
- 3차: 계정 또는 디바이스 제한

### 상태 전환 규칙
- `ACCEPTED`: 점수, 시간, 입력 빈도, 체크포인트 해시가 모두 허용 범위 내인 결과
- `FLAGGED`: 허용 범위를 일부 벗어나지만 운영 검토가 필요한 결과
- `REJECTED`: 체크포인트 해시 불일치, 명백한 불가능 시간, 필수 필드 훼손처럼 즉시 무효화 가능한 결과

### 정량 판정 기준 참조
- 상세 수치 기준은 `02_시스템설계서.md`의 부정행위 검증 정량 기준과 `12_게임규칙_수치테이블.md`를 따른다.

### 완료 조건
- 의심 기록이 자동 플래그되고 일반 랭킹에서 제외된다.
