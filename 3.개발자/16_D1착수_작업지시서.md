# D1 착수 작업지시서

## 1. 목적
- 본 문서는 `D0 완료` 이후 즉시 착수해야 할 D1 범위 작업을 개발팀에 지시하기 위한 실행 문서다.
- 기준 문서는 `2.설계문서`와 `3.개발자/15_설계대비_개발진행현황_리포트.md`다.

## 2. D1 목표
- 브라우저에서 게스트 사용자가 한 판을 정상 플레이할 수 있어야 한다.
- 게임 종료 후 결과 화면이 표시되고 즉시 재도전할 수 있어야 한다.
- 코어 플레이 흐름을 수동 및 자동 검증 가능한 상태로 만들어야 한다.

## 3. D1 범위
### 포함
- 게임 화면 기본 레이아웃
- 캔버스 또는 동등한 플레이 표면 렌더링
- 블록 스폰, 이동, 회전, 드롭, 충돌, 라인 제거
- 점수, 라인 수, 레벨, 다음 블록, Hold, Ghost Piece 기본 반영
- 게임 오버 판정
- 결과 화면
- 재도전 버튼과 세션 재시작 흐름
- 핵심 `data-testid`

### 제외
- 랭킹 반영
- 닉네임 등록
- 결과 제출 API
- 공유
- 운영 기능
- Daily Challenge 실동작 분기

## 4. 작업 우선순위
### 4.1 1순위
- `GE-002` 게임 엔진 D1 완료 수준까지 마무리
- `FE-002` 결과 화면 및 재도전 구현
- 게임 화면 라우트 및 상태 전이 구현

### 4.2 2순위
- HUD 최소 정보 노출
- Hold, Ghost Piece, 다음 블록 5개 표시
- 기본 입력 체계 연결

### 4.3 3순위
- 튜토리얼 placeholder 또는 스킵 가능한 최소 안내
- 포커스 아웃 시 일시정지 기본 동작
- 모바일 기본 버튼 배치

## 5. 세부 지시
### 5.1 게임 엔진
- `12_게임규칙_수치테이블.md` 기준으로 아래 항목을 구현한다.
  - 스폰 좌표
  - 회전 전이
  - 벽차기
  - 라인 제거
  - 게임 오버 판정
  - 레벨별 중력 기본값
  - 소프트 드롭/하드 드롭 점수
- D1에서는 최소한 Marathon 단일 모드만 정상 동작하면 된다.
- DAS/ARR, 락 딜레이는 설계 수치 기준으로 맞추되, 미세 튜닝은 D1 후반에 허용한다.

### 5.2 프론트엔드
- 게임 시작 시 `/` 랜딩에서 실제 플레이 화면으로 진입 가능해야 한다.
- 아래 요소를 첫 구현 범위에 포함한다.
  - 보드
  - 점수
  - 제거 라인 수
  - 레벨
  - 다음 블록 5개
  - Hold 블록
  - 재도전 버튼
- 결과 화면의 첫 CTA는 반드시 `재도전`으로 둔다.
- 결과 화면은 게임 종료 후 즉시 전환되게 구현한다.

### 5.3 테스트 가능성
- 아래 선택자를 반드시 부여한다.
  - `game-board`
  - `score-value`
  - `lines-value`
  - `level-value`
  - `next-queue`
  - `hold-slot`
  - `result-screen`
  - `retry-button`
- 게임 상태 전이는 테스트에서 관찰 가능해야 한다.
- 최소 1개 E2E 시나리오를 D1 범위에 추가한다.

## 6. 완료 기준
- 랜딩에서 `바로 시작` 클릭 후 게임 화면 진입 가능
- 키보드 입력으로 좌우 이동, 회전, 소프트 드롭, 하드 드롭 가능
- 라인 제거 시 HUD 값 갱신
- 게임 오버 또는 종료 조건 시 결과 화면 노출
- 결과 화면에서 `재도전` 클릭 후 새 판 시작 가능
- 엔진 단위 테스트가 설계 수치 기준으로 보강됨
- D1 핵심 흐름 E2E 1건 이상 추가됨

## 7. 산출물
- 게임 화면 구현
- 결과 화면 구현
- 엔진 보강 코드
- 엔진 테스트 추가
- E2E 시나리오 추가
- 작업관리대장 상태 갱신

## 8. blocker 기준
- 스폰/충돌/회전 규칙이 설계 문서와 다르게 동작
- 결과 화면 없이 게임 종료만 되는 상태
- 재도전 후 세션 재시작이 불가능한 상태
- 테스트 선택자가 누락돼 자동화 착수가 불가능한 상태

## 9. D1 종료 시 설계자 검토 포인트
- 게임 규칙이 `12_게임규칙_수치테이블.md`와 충돌하지 않는가
- 실제 플레이 흐름이 `F-02`, `F-04` 완료 조건을 최소 충족하는가
- D2 착수를 위해 모드 분기 구조가 확장 가능하게 설계됐는가
