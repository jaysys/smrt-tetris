# 개발자전달 캐주얼게임 UI강화요청서

## 1. 목적
- 본 문서는 현재 화면 구현에 `캐주얼 게임 서비스 요소`를 반드시 반영하도록 개발자에게 전달하는 요청서다.
- 목적은 단순 기능 배치형 UI를 넘어, 사용자가 `지금 누르고 싶고`, `다시 하고 싶고`, `보상을 기대하게 만드는` 화면으로 전환시키는 것이다.

## 2. 전달 기준 문서
- `23_UI화면설계서.md`
- `24_개발자전달_UI화면설계_요약.md`
- `25_기획자_화면설계개선의견_설계자회신.md`

## 3. 핵심 전달 사항
- 현재 화면은 기능 검증용 구조는 갖추고 있으나, 흥행형 캐주얼 게임 서비스 화면 톤으로는 부족하다.
- 따라서 이번 UI 구현은 `정리된 정보 화면`을 만드는 작업이 아니라, `플레이 충동`, `재도전 충동`, `보상 기대`, `경쟁 감각`을 화면에 넣는 작업으로 이해해야 한다.
- 특히 아래 4가지는 반드시 반영해야 한다.
  - 강한 Hero
  - 보상 언어
  - 게임다운 조작감 시각화
  - 살아 있는 서비스 톤

## 4. 캐주얼 게임 요소 반영 요구
### 4.1 랜딩
- 첫 화면은 카드 목록이 아니라 `지금 한 판 시작하고 싶게 만드는 Hero 화면`이어야 한다.
- `바로 시작`은 가장 강한 1등 CTA여야 한다.
- Hero에는 아래 중 최소 1개가 들어가야 한다.
  - 마스코트 또는 감정 표현 오브젝트
  - 오늘의 챌린지를 소개하는 쇼케이스
  - 최근 성과/최고 기록 갱신을 상징하는 보상 장면
- `최근 성과`는 단순 수치 나열이 아니라 배지, 불꽃, 스탬프, 갱신 마크 같은 보상 언어로 보여야 한다.

### 4.2 모드 선택
- Marathon, Sprint, Daily Challenge는 설명 카드가 아니라 `들어가고 싶은 포스터`처럼 보여야 한다.
- 각 모드는 서로 다른 감정 톤을 가져야 한다.
  - Marathon: 오래 버티는 생존/성장감
  - Sprint: 속도 경쟁과 기록 압박감
  - Daily Challenge: 오늘의 보상과 미션 감각
- 모드 카드에는 반드시 아래가 들어가야 한다.
  - 대표 비주얼
  - 한 줄 감정 카피
  - 핵심 목표
  - 경쟁 또는 보상 포인트

### 4.3 플레이 화면
- 플레이 화면은 `보드가 주인공`이어야 한다.
- 보드 바깥 정보 패널은 폼처럼 보이면 안 된다.
- 점수, 콤보, 레벨은 상승하고 있다는 감각이 느껴져야 한다.
- 조작 버튼은 일반 웹 버튼이 아니라 `게임패드형` 감각을 가져야 한다.
- 배경, 그리드, 라이트, 입자감 등으로 긴장감을 보강하되, 보드 가독성을 해치면 안 된다.

### 4.4 결과 화면
- 결과 화면은 `성과 확인 화면`이 아니라 `다시 도전하게 만드는 보상 화면`이어야 한다.
- `재도전`은 화면에서 가장 강한 CTA여야 한다.
- 첫 3판 개인 최고 기록 갱신 시 일반 결과와 완전히 다른 축하 상태를 가져야 한다.
- 결과 Hero는 아래 중 최소 1개를 크게 보여야 한다.
  - 최종 점수
  - 최고 기록 갱신
  - 랭킹 진입
- 공유 CTA는 단순 기능 버튼이 아니라 `자랑하고 싶게 만드는 버튼`이어야 한다.

### 4.5 랭킹 화면
- 랭킹은 표 조회 화면이 아니라 `경쟁 무대`처럼 보여야 한다.
- Top 3는 일반 리스트와 다르게 시상대/메달/트로피 구조를 가져야 한다.
- 내 기록은 별도 강조 카드로 분리해야 한다.
- 순위, 상위 퍼센트, 오늘 내 최고 위치 같은 비교 언어를 강화해야 한다.

### 4.6 Daily / 보상 / 닉네임 등록
- Daily Challenge는 설명 시트가 아니라 `오늘의 보상 안내판`처럼 보여야 한다.
- 닉네임 등록은 단순 입력폼이 아니라 `공식 기록에 이름을 새긴다`는 감정을 줘야 한다.
- 보상, 배지, 기록 저장은 모두 `수집 욕구`를 자극하는 오브젝트 표현을 가져야 한다.

## 5. 반드시 피해야 할 구현
- 화면 대부분을 동일 톤 카드 박스로 반복하는 방식
- 플레이 화면을 설정 패널처럼 보이게 만드는 방식
- 랜딩 Hero 없이 정보 카드만 나열하는 방식
- 결과 화면을 점수 박스와 버튼 반복으로 끝내는 방식
- 기술 검증용 문구, 임시 영문 제목, 내부 상태 설명을 사용자 UI에 남기는 방식
- 라이선스 미확보 상태에서 공식 테트리스 트레이드드레스처럼 보이게 만드는 방식

## 6. 최소 구현 완료 기준
- 사용자가 첫 3초 안에 `바로 시작`, `오늘의 챌린지`, `최근 성과` 중 최소 2개를 강하게 인식할 수 있어야 한다.
- 랜딩, 모드 선택, 결과, 랭킹은 각각 서로 다른 감정 톤을 가져야 한다.
- 모바일 세로에서도 `게임형 화면 밀도`가 유지되어야 한다.
- 결과 화면은 정보 확인보다 재도전 유도가 더 강해야 한다.
- 화면이 `문서형 정보 앱`처럼 보이지 않고 `캐주얼 게임 서비스`처럼 보여야 한다.

## 7. 개발자 액션 요청
1. 현재 화면을 기준안으로 유지한 채 미세 수정하지 말고, Hero/카드 계층/보상 언어부터 다시 잡는다.
2. 랜딩, 모드 선택, 결과, 랭킹은 전면 재설계 대상으로 본다.
3. 플레이 화면은 보드 중심 구조는 유지하되, 패널의 폼 느낌을 제거한다.
4. 마스코트 또는 감정 표현 오브젝트, 배지/메달/스탬프 체계를 포함한 독자 IP 시각 언어를 반영한다.
5. 최소한 `랜딩 Hero`, `결과 Hero`, `랭킹 Top 3`, `Daily 보상 카드`에는 캐주얼 게임 요소가 분명히 드러나야 한다.

## 8. 최종 전달 문안
- 이번 UI 수정은 단순 정리 작업이 아니다. 화면에 캐주얼 게임 서비스 요소가 반드시 반영되어야 한다. 기존 화면은 기능 구조는 있으나 게임다운 감정, 보상 기대, 경쟁 무드, 재도전 충동이 약하다. 따라서 앞으로는 `23_UI화면설계서.md`, `24_개발자전달_UI화면설계_요약.md`, `25_기획자_화면설계개선의견_설계자회신.md`를 기준으로, 화면을 `정보 카드형 웹`이 아니라 `흥행형 캐주얼 게임 서비스 UI`로 다시 구현해 달라.
- 특히 랜딩 Hero, 모드 포스터, 플레이 화면의 게임패드형 조작부, 결과 Hero, 랭킹 Top 3, Daily 보상 카드에는 캐주얼 게임 요소가 분명히 보여야 한다. 배지, 불꽃, 스탬프, 메달, 보상 오브젝트, 감정 표현 오브젝트를 적극 활용하되, 공식 테트리스 트레이드드레스를 모사하지 않는 독자 IP 방향으로 구현해 달라.
